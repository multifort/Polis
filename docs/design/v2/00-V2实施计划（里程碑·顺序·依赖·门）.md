## 0. 目标与定位
V2 = 从 V1「能转」到「**能协同**」：让目标**驱动结构本身**（检索/生成对的计划与团队）、跑得稳、错能修、长能跑、并发可控，并让资产**自增长**（越用越聪明）。V1 验收门已用数据证明闭环可行（DAG/路由 100% · 人审 91.7% · 成本时延在预算），具备进 V2 资格。

## 1. 排序原则
1. **地基先行**：被多块依赖的（检索底座 R1、任务黑板 B1、任务实体 P1）先做。
2. **生成内核为脊柱**：阶段 A 解除"模板固定"天花板，是 V2 核心价值。
3. **每阶段一个数据门**：跑数据决定是否进下一阶段；**未达标→回炉调检索/生成/prompt/记忆/eval，不堆模块**（F3/F9）。
4. **复用优先**：能靠 Temporal/pgvector/LiteLLM/Evaluator 搭的不自研。
5. **风险前置**：RLS 演进、MinIO、LLM 生成可靠性各有兜底与门兜底。

## 3. 跨工作流依赖图
```
R1(检索底座) ─┬─▶ A1(检索) ─▶ A2(规划生成) ─▶ S2(纠错/重规划) ─▶ A4(背书) ─▶ R3(场景沉淀=飞轮)
              └─▶ R2(角色模板) ─▶ A3(编配生成) ─▶ R4(角色自动沉淀)
B1(任务黑板) ─┬─▶ S1(状态机+质量门) ─▶ S3/S4(并发/长跑)
              └─▶ B2(分层接地) ─▶ B3/B4(晋升/预算)
P1(任务实体) ─▶ P2(附件+MinIO) ─▶ P3(执行记录+导出) ─▶ P4(看板) ;  P5(场景分类)←R3
```
**关键路径**：`R1 → A1 → A2 → S2 → A4 → R3`（飞轮闭环）。
**并行轨**：① 记忆 `B1→B2→B3/B4`；② 产品 `P1→P2→P3→P4`；③ 协同 `S1→S2/S3/S4`。三轨与内核轨交汇于各阶段门。

## 4. 阶段化里程碑

### 阶段 0 · 地基（低耦合、可并行、为后续供能）
**目标**：打好"检索 / 执行 / 记录"的地基。
| 里程碑 | 内容 | 依赖 |
|---|---|---|
| **R1** 检索底座+可见性 | skill/plan_template 加 `visibility`/`embedding`/hnsw；**RLS 读放行 public**；内置资产置 public | M6 embed✓ / RLS✓ |
| **B1** 任务黑板 | `result_envelope.content` + `read_node_output`/`query_task_memory` + 依赖摘要注入（彻底修 F3） | M4/M5✓ |
| **P1** 任务实体+列表 | 新建 `task` 表；`task_run.task_id`；任务列表/可复用/重跑 | — |
| **S1** 状态机+质量门 | 节点状态机（needs_rework/replanning）+ 关键节点 Evaluator 检测 | Evaluator✓ + B1 |

**门**：黑板下游确定性拿到上游(协同可靠性 100%)；任务可多次运行留多条记录；关键节点质量门可触发。
**并行**：R1/B1/P1 可并行；S1 紧跟 B1。

### 阶段 A · 生成式内核（脊柱，解除模板天花板）
**目标**：目标驱动结构——检索不到就生成，且生成必过确定性校验。
| 里程碑 | 内容 | 依赖 |
|---|---|---|
| **A1** 检索升级 | 模板/角色/技能/能力**语义检索**（替换关键词/遍历） | R1 |
| **A2** 规划生成 | RAG 接地 LLM 生成 DAG + 双校验 + 有界自修复（**偿还 TD-020**） | A1 |
| **R2** 角色模板层 | `role_template` 表；preset 改引用；编配从 role_template 选/拼 | R1 |
| **A3** 编配生成 | 拼 Skill 成 Agent（自动）+ 缺 Skill 生成草稿 + 人审入口 | A1+B1+R2 |
| **B2** 分层接地 | retrieve 按消费者分层（org→规划 / role+org→Agent）+ 预算截断 | B1+A1 |

**门**（扩 M7 目标集到"模板外/新维度目标"再跑）：无模板目标 DAG 可用率 ≥ 70%；生成 DAG 结构合法率 ≥ 80%；编配覆盖率 ≥ 90%。

### 阶段 B · 稳态与质量（跑得稳 · 错能修 · 长能跑 · 贵能控）
**目标**：把"动态/生成的"计划跑得可靠。
| 里程碑 | 内容 | 依赖 |
|---|---|---|
| **S2** 分级纠错+局部重规划 | retry→rework→local replan(最小优先)→escalate→abort + 重 validate 拼接 | S1+A2 |
| **A4** 自动背书 | Evaluator 三件套常态化 + 热路径阈值闸 + 沉淀回写 | A2/A3 |
| **B3/B4** 晋升+预算治理 | task→role→org 自动晋升；上下文 token 预算 enforce | B1/B2 |
| **S3/S4** 并发+长跑 | org 并发上限 + 预算提示（不阻断）；workflow versioning + heartbeat | S1 |

**门**：自动纠错成功率 ≥ 60%；任务完成率 ≥ 90%；长任务可恢复 100%；并发无串扰 100%；人审通过率再升。

### 阶段 C · 产品化与飞轮（用得起来 + 自增长闭环）
**目标**：产品主线打通 + 资产飞轮起效。
| 里程碑 | 内容 | 依赖 |
|---|---|---|
| **P2/P3** 附件+MinIO / 执行记录+导出 | MinIO 接入；附件注入；执行记录详情（观测页）+ md/pdf 导出 | P1+MinIO |
| **P4** 看板 | 跨任务/场景运营统计 + 实时过程观测 | P3+S3 |
| **R3/R4/P5** 飞轮 | 满意→场景模板沉淀（大类>小类>场景）；生成 Agent→角色模板自动沉淀；场景库树导航 | R1/R2/A4/P3 |

**门**：历史可追溯 100%；附件/导出端到端通；**复用命中率↑（飞轮起效）**；自动沉淀有效性 ≥ 60%。

## 6. 前置 / 风险与缓解
| 项 | 影响 | 缓解 |
|---|---|---|
| **RLS 读放行演进**（R1） | 动多租户根基 | 与 R1 同步扩 T8.3 隔离回归；写策略不放宽；放宽只在读、只放行 public |
| **MinIO**（P2 前置） | 新基础设施 | docker-compose 加一服务；key 按 `{org_id}/{task_id}/` 前缀隔离 |
| **LLM 生成可靠性**（A2） | V2 最大技术风险 | RAG 接地 + 结构/语义双校验 + 有界自修复 + 阶段门兜底 |
| **成本上升** | 生成/eval 增 LLM 调用 | 每阶段守成本门（F3 已证敏感）；检索命中率越高越省 |
| **worker 代码变更** | 在飞任务 | S4 workflow versioning；运维上"改 worker 必重启"（沿用 V1 纪律） |
