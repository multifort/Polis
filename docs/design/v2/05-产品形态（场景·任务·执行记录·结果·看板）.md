## 0. 一句话
把内核能力串成用户主线——**建公司 → 配角色 →（可选）攒场景 → 建任务（带附件）→ 跑 → 看执行记录 → 取结果 → 看看板**；任务可复用、每次执行可追溯、结果可导出。

## 1. 背景与目标

### 1.1 V1 产品现状（缺口）
- 只有"任务/计划清单"，**看不到历史**（每次出图即覆盖）。
- **交互入口缺失**：如采购分析需上传附件材料，**无上传入口**。
- 执行结果**无法导出/移植**（只有屏上文本）。

### 1.2 目标
1. **任务列表（任务可复用）** + **执行记录**（每次执行：过程 + 结果 + 下载）。
2. 补**附件输入入口**与**结果导出**。
3. 产品主线串联（§3 的 1–7 步）+ **Dashboard/过程观测/运营统计**。

### 1.3 非目标（后置）
- Word/Excel 导出（先 md+pdf）；跨组织共享展示（V3）。

## 2. 信息架构（IA）与主线
```
公司(Org) ─ 角色(Role/花名册) ─ 场景库(Scenario·大类>小类>场景) ─ 任务(Task·可复用)
                                                                      │ 运行
                                                                      ▼
                                                          执行记录(Run) ── 结果产物(Artifact·可下载)
                                                                      │
                                                                Dashboard(运营统计/过程观测)
```
用户主线（你列的 1–7，已自洽）：
1. 建公司 → 2. 配角色 → 3.（可选）攒场景 → 4. 建任务（选/生成场景 + 传附件）→ 5. 跑并记录 → 6. 看结果/导出 → 7. 看看板。

> **3 与 4 不矛盾**：场景=模板层（可空、可自动沉淀），任务=实例层（日常入口）。详见本文 §3（与 [仓库文档](04-资产仓库（三层·可见性·复用·晋升）.md) §1.3 的"场景模板≠公司蓝图"澄清一致）。

## 3. 对象模型：场景 / 任务 / 执行记录（三层）
```
场景模板(Scenario)   "怎么做这类事"   可复用模板，可空、可自动沉淀；按 大类>小类>具体场景 归类
   │ 1:N（可选绑定）
任务(Task)           "我要办的这件事" 具体、带输入/附件、可保存可反复跑
   │ 1:N
执行记录(Run)        "这次办的过程+结果"  trace + 结果产物 + 下载
```
- **任务为主**（用户决策）：日常建任务；任务在内核里**检索命中场景就用、没有就生成**，无需先定义场景。
- **场景自动沉淀**：满意任务 → 沉淀为场景模板（飞轮，仓库文档 §6）。
- **场景分层归类**（用户决策，防平铺堆砌）：**大类 → 小类 → 具体场景**（如 采购 > 供应商管理 > 供应商交付分析）。

## 4. 数据模型（向后兼容 · 先加后删）

| 表 | 处理 | 字段 |
|---|---|---|
| **`task`（新）** | 任务=独立可复用实体（用户决策 fork1） | id/org_id/name/`scenario_ref`(plan_template,nullable)/goal/`input_schema`(JSONB)/`inputs`(JSONB 默认)/created_by/status/created_at |
| `plan` | **退为"某次运行的 DAG 快照"** | 现字段不变；由"任务运行时"生成快照（不再是用户直接持有的实体） |
| `task_run` | 复用 + 关联任务 | + `task_id`(FK→task, nullable 兼容旧)；已有 plan_id/status/started/finished |
| `result_envelope` | 复用（执行记录的产出） | 见记忆文档（+ content/tokens） |
| **`artifact_descriptor`** | 复用为**输入附件 + 结果产物** | 已有 modality/uri/mime/caption/meta + task_id/node_id；`uri` 指向 MinIO |
| `plan_template`（场景模板） | + **分类字段**（见 §归类） | + `domain`(大类) + `subcategory`(小类)（name=具体场景）；并回填仓库文档 04 |

迁移：新建 `task`；`task_run.task_id` 加列(nullable)；旧 plan/run 仍可用（task_id 空=临时/历史）。

### 场景归类（大类>小类>具体场景）
- 最小实现：`plan_template.domain`（大类，对齐 `capability.domain`）+ `subcategory`（小类）+ `name`（具体场景）。
- 若需更深嵌套：可换 `scenario_category` 自引用树表（parent_id），现不必，留扩展位。
- 用途：场景库**按树导航**、检索按 大类/小类 过滤、避免平铺。

## 5. 输入 / 附件入口（补交互缺口）
```
场景/任务声明 input_schema（需要哪些输入/附件，如"供应商报价单.xlsx"）
   ▼ 用户在建任务时上传
文件 → 对象存储(MinIO) → artifact_descriptor(modality='file', uri, mime)
   ▼ 任务运行时
作为任务输入注入上下文（经记忆黑板：节点按需 read，记忆文档 §7）
```
- 附件是**任务输入**，按需注入（不全量塞上下文，受 token 预算约束）。
- `input_schema` 由场景声明（生成/沉淀时带出），任务实例填具体值/文件。

## 6. 执行记录（#6）
- **任务列表**（可复用）：org 下所有任务，可再次运行。
- **执行记录列表**：每任务的历次 `task_run`（状态/时间/成本/结果）。
- **单次记录详情**：**复用已建的"运行观测页"（M6-H-4）**——DAG 过程 trace + 节点产出 + 用量 + 完整日志。
- **重跑/重放**：基于任务重新运行（新建一条 Run）；保留历史不覆盖。

## 7. 结果存储与展示（#7）
- **产物模型**：结果存为 `artifact_descriptor`（report.md、export.pdf...），`uri`→MinIO。
- **多格式渲染/导出**：**先 Markdown + PDF**（足够"可移植"）；**Word/Excel 后置**（Excel 适合比价表等结构化结果）。
- **展示**：观测页/结果页渲染 markdown（已有 remark-gfm）；提供**下载**按钮（取 artifact uri）。
- 导出 = 渲染结果 → 生成文件 → 落 artifact → 下载链接。

## 8. Dashboard / 运营统计（7）
- **跨 task_run 聚合**：成本（¥，按目录价实算，已有）/ token / 成功率（done vs failed）/ 平均时延 / 吞吐 / 按场景·按角色分布。
- **过程观测**：进行中任务的实时状态（接协同框架并发视图）。
- **运营统计**：预算使用（仅提示，协同 §6）、复用命中率（飞轮指标）、人审通过率（eval）。
- 复用：扩展已建的 observability `totals/by_model` 聚合到"跨任务/跨场景"。

## 9. 对象存储（MinIO）引入（用户决策 fork2）
- docker-compose 增 **MinIO**（V2 路线既定"对象存储换 MinIO"）。
- `artifact_descriptor.uri` = `s3://{bucket}/{org_id}/{task_id}/...`；**按 org 前缀隔离**。
- 附件上传 + 结果导出走 MinIO；凭证走 env（不入库不入日志）。
- 不引其它新组件（复用优先）。

## 13. 关键决策（建议落 ADR-0014）
- **任务=独立可复用实体**（新 `task` 表）；plan 退为运行 DAG 快照（用户决策）。
- **引入 MinIO** 作对象存储（附件 + 结果）（用户决策）。
- 导出**先 md+pdf，Word/Excel 后置**（用户决策）。
- **任务为主入口**，场景可选/自动沉淀，且**按大类>小类>具体场景归类**（用户决策）。

## 14. 未决问题
1. `input_schema` 的表达力（仅文件 vs 结构化参数 vs 二者）；校验时机。
2. 任务"重跑"是否允许改输入（同任务不同输入 = 新 Run 还是新 Task？）。
3. 大类/小类的词表是受控还是自由（建议受控、对齐 capability.domain，防平铺/重复）。
4. 导出渲染器选型（md→pdf：wkhtmltopdf/weasyprint 等，复用优先选轻量）。
