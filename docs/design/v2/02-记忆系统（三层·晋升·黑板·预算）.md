## 0. 一句话
记忆分**公司/角色/任务**三层（作用域），**一切按需拉取（pull）不推全链路**；任务级原始记录自动**蒸馏为事实并逐层晋升**，让公司越用越懂自己——且上下文 token 始终**受预算约束**。

## 1. 背景与目标

### 1.1 V1 现状（M5/M6）
- `memory` 表：`scope ∈ (session,task,role,org)`、`type ∈ (factual,procedural,preference,event)`、`content`/`embedding`(1024)/`provenance`/`importance`/`confidence`/`decay_rate`/`expires_at`，含 hnsw 向量索引。
- `result_envelope`：任务产出（`task_id`/`node_id`/`agent_id`/`status`/`artifacts`/`facts`/`summary`/`needs_human`）——**已是任务级原始记录**。
- 管线（`memory/center.py`）：`write_facts`（标准化→去噪→去重→评分→embed→入库）、`retrieve`（作用域过滤+相关性+importance/recency+`MemorySlice.to_text()` 注入摘要）、`upsert_shared_fact`（org 并发裁决：insert/override/conflict）、`decay`（衰减遗忘）。
- 检索：M6-D 已切**向量 RAG**（bge 1024）。

### 1.2 V1 实证（F3 发现的关键缺陷）
- **下游节点看不到上游产出**：`context.build` 只把 `node.input_hint` + 模糊检索的记忆喂给 Agent；n4"整合前面分析"全靠**碰运气式向量检索**召回——不可靠。
- 即"任务内节点协同"目前**没有可靠的数据通路**。

### 1.3 V2 目标
1. 把 org/role/task **三层正式化**，明确"存什么、谁读、怎么衰减"。
2. **任务内可靠协同**：任务黑板 + 按 id 懒加载（解决 F3 缺陷），且**token 受控**。
3. **自动晋升**（task→role→org）实现"自增长"，无人 curate（用户决策）。
4. **跨任务共享**：靠晋升 + 按需 RAG，**从不传递、只拉取**。

### 1.4 非目标（后置）
- 跨组织共享记忆（V3）。
- 对象存储/大产物（归 #7 多类型结果，用 `artifact_descriptor`，本块不展开）。

## 2. 设计总纲：一切皆"按需拉取"

```
唯一被线程传递的 = 轻量 { goal + 依赖节点引用 + 一行摘要 }
全文/事实        = 永远从"分作用域共享存储"按需 pull
```

两类共享存储：
- **任务内（节点间）= 任务黑板**：节点产出写进 `result_envelope`（task 作用域）；下游**按 id 懒加载**全文，或在 task 作用域内检索。
- **任务间（多任务）= org/role 两层**：任务蒸馏 fact → **自动晋升** → 别的任务**按需 RAG 检索**。

→ 上下文大小与 DAG 规模**解耦**：不管多深多宽，默认进上下文的都是小摘要 + 限额 top-k，全文只在 LLM 明确需要时才取。

## 3. 三层（四作用域）模型

| 作用域 | 定位 | 存什么（粒度） | 主要消费者 | 生命周期 |
|---|---|---|---|---|
| `org`（公司级） | 实时查询的"公司知识" | **蒸馏 fact**：供应商画像/领域常识/历史结论；高置信+出处 | **规划器/编配器接地**、跨任务 | 长期·缓衰减·可 deprecate |
| `role`（角色级） | 按上下文处理 | **蒸馏 fact**：岗位经验/惯例/偏好/套路 | 该角色 Agent 推理 | 中期·随用强化 |
| `task`（任务级） | 任务历史 + 节点间协同 | **原始 envelope**（产出链）+ 临时 fact | 任务内协同、历史回看(#6) | 短期·归档进历史 |
| `session`（会话级，已存在） | 单次交互上下文 | 临时对话态 | 交互 | 极短·随会话清 |

区分：**org/role 存"蒸馏 fact"（`memory` 表）；task 存"原始记录"（`result_envelope`）+ 必要的临时 fact**。

> 扩展位（自增长）：`scope` 为开放枚举，预留 `team`（若 Role:Agent 转 1:N）与 `global`（V3 跨组织），现不实现。

## 4. 数据模型（向后兼容 · 先加后删）

| 表 | V1 现状 | V2 增量 | 迁移 |
|---|---|---|---|
| `memory` | scope/type/content/embedding/importance/confidence/decay_rate/provenance/expires_at | + `promoted_from UUID null`（晋升来源 envelope/fact 溯源）<br>+ `last_promoted_at` | 加列，旧行 null |
| `result_envelope` | task_id/node_id/agent_id/status/artifacts/facts/summary/needs_human | + `content TEXT null`（产出**全文**，供黑板懒加载；现仅 summary）<br>+ `tokens INT null`（产出 token 数，预算用） | 加列；旧行 content 回填自 facts/summary |
| `artifact_descriptor` | modality/uri/mime/caption/meta | （归 #7）大产物落对象存储后写 uri | 无 |

> 说明：org/role/task 三作用域 + 4 类 type + embedding/出处/衰减 **V1 schema 已具备**。V2 的核心增量是 `result_envelope.content`（让黑板能按 id 取全文）+ 晋升溯源字段，DDL 很轻。

## 5. 写入与晋升（自增长引擎）

### 5.1 写入（沿用 `write_facts`）
任务执行中：
- 节点产出 → 写 `result_envelope`（summary + **content 全文** + facts + tokens）。
- 从产出抽取的 fact → `write_facts(scope='task', ...)`（标准化→去噪→去重→评分→embed→入库）。

### 5.2 层间自动晋升（task→role→org）— 无人 curate
```
晋升候选 = task 层 fact，满足：
   importance ≥ θ_imp   AND   confidence ≥ θ_conf   AND   出现频次 ≥ θ_freq(跨节点/跨任务)
   AND 不是噪声(_is_noise)   AND 语义去重后非已有(find_similar)

task→role：达 θ_role 阈值 → write_facts(scope='role', namespace=角色, provenance 带 promoted_from)
role→org ：达 θ_org（更高，且 confidence≥θ_org_conf 且 provenance 必含来源）
          → upsert_shared_fact(scope='org')  # 复用并发裁决：insert/override/conflict
```
- **公司级置信门更高 + 必带出处**（防把幻觉沉淀成"公司知识"）。
- 晋升是**数据操作**（非代码），风险低 → **自动、无人审**（用户决策；不理想再加 curate）。
- **与内核飞轮同源**：记忆晋升 ∥ 模板晋升 ∥ Skill 沉淀 = 同一飞轮三面。

### 5.3 衰减/遗忘（分层 `decay`）
| 作用域 | TTL/衰减 |
|---|---|
| session | 极短（会话结束清） |
| task | 短（任务完成后 envelope 归档进历史#6，临时 fact 设 `expires_at`） |
| role | 中（`decay_rate` 缺省，访问刷新 `last_accessed`） |
| org | 长（低 `decay_rate`；可人工 `deprecate`） |
- 复用现有 `decay_job`，改为**按 scope 取不同 decay_rate / TTL**。

### 5.4 晋升/背书与信任的一致性
org 级 fact 作为"被当真相查询"的知识，其置信来自：高 confidence + 必有 provenance + 通过去重裁决。**不引入额外人审**，但保留 `conflict` 标记交人裁决（已有机制）。

## 6. 读取 / 接地（RAG，按层分工）

### 6.1 `retrieve` 分工
| 消费者 | 读哪层 | 用途 |
|---|---|---|
| 规划器/编配器 | **org**（+ 模板仓库） | "公司知道什么/干过什么相似的" = 检索退化链的先验 |
| Agent 推理 | **role + org** | 岗位套路 + 公司知识 |
| 任务内节点 | **task 黑板** | 上游产出（见 §7） |

- 默认 Agent 读 `scopes=['role','org']`（V1 已是），**不串读别的角色**（聚焦/隐私）。
- 全部 `org_id` + RLS 隔离。

### 6.2 注入摘要非全量
沿用 `MemorySlice.to_text()`：检索注入的是**摘要 + 出处**，不是全文（全文按需 §7）。

## 7. 任务内协同总线（黑板 + 懒加载）— 解决 F3 缺陷

### 7.1 机制
```
节点执行后：write_envelope(task_id, node_id, summary, content=全文, facts, tokens)
下游节点上下文（默认，便宜）：
   goal(小) + 直接依赖[{node_id, 一行摘要}](小) + role/org 检索 top-k(限额) + task 黑板检索 top-k(限额)
下游需要某上游细节（按需，贵）：
   调内置确定性技能 read_node_output(node_id) → 从 result_envelope.content 拉全文
```
- **可靠**：按 `node.deps` 的 `node_id` **精确取**依赖（不靠模糊检索碰运气）。
- **省**：默认只给摘要；全文懒加载，只付实际读取部分。
- **兜底**：直接依赖的**摘要一定自动带上**（保证下游知道上游产出了什么）。

### 7.2 取数=确定性技能，决策=LLM（契合分工线）
新增两个**内置 MCP 本地工具**（与 echo/calc 同类，确定性）：
| 技能 | 签名 | 作用 |
|---|---|---|
| `read_node_output` | `(node_id) -> {summary, content, provenance}` | 按 id 拉某上游节点全文 |
| `query_task_memory` | `(query, k) -> [snippet]` | 在 task 作用域内语义检索 |
| `query_memory` | `(query, scopes, k) -> [snippet]` | 跨 role/org 按需检索 |

- LLM 决定"要不要拉、拉哪个"；拉取动作确定性、可审计（计入 `skill_invocation`）。

### 7.3 与内核/#5 的衔接
- 内核执行节点时，`context.build` 注入"goal + dep 摘要 + 限额检索"，并把上述工具暴露给 Agent。
- 任务态可靠 → #5 协同框架才能"检测错误/局部重规划"（重跑节点时黑板提供已完成上游）。

## 8. token 预算治理

### 8.1 上下文预算公式
```
ctx_tokens(node) ≈ |goal| + Σ|dep 摘要| + k_mem·|摘要| + k_task·|片段|
受约束：ctx_tokens ≤ B_node（每节点预算）
策略：k_mem/k_task 限额；摘要长度截断；超预算时按 importance/相似度截断
全文(read_node_output) 仅按需计入，单独计预算 B_fetch
```
- 直接挂 V1 验收门的"**成本/时延在预算**"。F3 已证产出变厚显著抬成本（¥0.0154→0.023），故预算必须 enforced。
- 观测：每节点上下文 token 入 `result_envelope.tokens` + Langfuse，便于调 k/B。

### 8.2 默认值（先定，按数据校准）
- `B_node` ≈ 模型上下文窗口的 25%；`k_mem=k_task=5`；摘要 ≤ 200 字；`read_node_output` 全文上限 4k tokens（超则返回其 summary + 提示分段取）。

## 9. 隔离与并发
- **多租户**：`memory`/`result_envelope` 均 `org_id` + RLS（已有）；跨租户零可见（M7-T8.3 回归覆盖）。
- **读权限**：Agent 默认 `role+org`，不读他角色；`task` 仅本任务。
- **并发写 org**：复用 `upsert_shared_fact`（insert/override/conflict），晋升到 org 走同一裁决。

## 12. 关键决策（建议落 ADR-0011）
- 三层（四作用域）+ org/role 存蒸馏 fact、task 存原始 envelope。
- **自动晋升、无人 curate**（org 置信门更高+必出处）；不理想再加 curate。
- **任务内：黑板 + 按 id 懒加载 + 限额检索**；任务间：晋升 + 按需 RAG；**从不推全链路**。
- 取数=确定性技能、决策=LLM；全程 token 预算受控。

## 13. 未决问题
1. 晋升阈值 θ_imp/θ_conf/θ_freq、θ_role/θ_org 初值（先经验值，按数据校准）。
2. `read_node_output` 全文上限与"长产出分段取"的策略。
3. role 层是否需要进一步按 namespace 细分（多任务下角色经验聚合粒度）。
4. session 层在 V2 是否启用（当前交互态是否需要持久），还是仍只用 task。

## 14. 验收指标（本块）
| 指标 | 定义 | 门槛 |
|---|---|---|
| 协同可靠性 | 下游能确定性拿到直接依赖产出（黑板命中） | 100% |
| 上下文预算达标 | 节点上下文 token ≤ B_node | ≥ 95% |
| 晋升精确率 | 晋升到 org 的 fact 人抽检"正确且有出处" | ≥ 90% |
| 检索贡献 | 命中 org/role 记忆使产出质量(judge)提升 | 正向（A/B 观测） |
