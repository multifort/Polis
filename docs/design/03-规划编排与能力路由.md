## 1. 职责边界（消除 Planner/Routing 重叠的关键决策）

```text
Planner   目标 → DAG；节点只声明 requiredCapabilities，【不绑定具体 Agent】
Validator DAG 校验(无环/完整/预算/能力可满足)
Router    执行派发时按能力需求选 Agent(运行时)
Temporal  执行 DAG(重试/暂停/补偿/有界重规划)
```

## 2. Planner 五级流水线（自研）

```text
Goal Understanding → Task Decomposer → Capability Tagging → DAG Builder → Plan Optimizer
```

### 2.1 各级接口
```python
class Planner:
    def understand(goal:str, ctx:OrgCtx) -> Intent          # LLM: {domain, intent, constraints}
    def decompose(intent:Intent) -> list[Task]              # LLM+模板: 子任务列表
    def tag_caps(tasks) -> list[TaskWithCap]                # 每任务标 requiredCapabilities(词表)
    def build_dag(tasks) -> Dag                             # 推断依赖(LLM+规则)
    def optimize(dag:Dag) -> Dag                            # 并行化 + 选执行器档/模型档
```

### 2.2 模板优先（早期靠模板兜可靠性）

def plan(goal, ctx):

## 4. ★ Plan 校验（拦截垃圾计划）

```python
def validate(dag, org) -> Result:
    assert is_acyclic(dag)                                  # 无环
    assert all_nodes_reachable(dag)                         # 完整可达
    for n in dag.nodes:
        assert RouterIndex.can_satisfy(n.requiredCapabilities, org)  # 能力可被满足
        if n.dangerous: assert n.type == "human" or n.gated          # 危险动作已标人审
    assert estimate_cost(dag) <= dag.budgetCents            # 预算内
# 不过 → 退回 Planner 重出(限 N 次) / 提示人工
```
通过后 →【人审：确认/改 DAG + 看预估成本】(审批收件箱 06) → 启动 Temporal。

## 5. Temporal 工作流/活动（集成细节）

### 5.1 Workflow 签名
```python
@workflow.defn
class TaskWorkflow:
    @workflow.run
    async def run(self, plan: Plan, org_id: str) -> Result:
        for node in topo_order(plan):
            await self._await_deps(node)
            if node.type == "human":
                await workflow.wait_condition(lambda: self.signaled[node.id])  # 人审挂起
                continue
            res = await workflow.execute_activity(
                run_node, args=[node, org_id],
                start_to_close_timeout=timedelta(minutes=10),
                retry_policy=RetryPolicy(maximum_attempts=3, backoff_coefficient=2))
            if not res.ok and res.replannable:
                await self._bounded_replan(node)              # §6
    @workflow.signal
    def approve(self, node_id: str): self.signaled[node_id] = True
```

### 5.2 Activity（节点执行 = 调 Router + Agent Runtime 04）
```python
@activity.defn
async def run_node(node: Node, org_id: str) -> NodeResult:
    agent = Router.select(node.requiredCapabilities, org_id)   # §7
    return await AgentRuntime.execute(agent, node, org_id)     # 见 04
```
- Activity 类型：Agent / Skill(直接 MCP) / System(DB/HTTP)；Human 在 workflow 内 signal。

## 6. ★ 有界动态重规划

```python
async def _bounded_replan(self, failed_node):
    if self.replan_count >= MAX_REPLANS: raise NeedHuman()    # 上限，防无限自主
    patch = await execute_activity(replan_local, args=[failed_node, self.context])
    # patch 仅允许: 重排/插入/替换 ≤ K 个节点；不允许整盘重来
    validate(apply(self.plan, patch))                          # 重规划也要过校验
    if patch.requires_approval: await self._await_approval(patch)
    self.replan_count += 1
```

## 7. 能力路由（自研差异化 · 详细）

### 7.1 流水线
```text
node.requiredCapabilities → CapabilityMatcher → RankEngine → AgentSelector
```

### 7.2 匹配 + 打分
```python
def select(req_caps, org_id) -> Agent:
    cands = AgentIndex.by_org_capability(org_id, req_caps)     # SQL: agent_capability ∩
    scored = []
    for a in cands:
        cap   = cosine(embed(req_caps), embed(a.capabilities)) # 能力 embedding 匹配
        hist  = success_rate(a)            # 来自 run_manifest/eval(06)，冷启动=None
        cost  = norm_cost(a.model)
        lat   = norm_latency(a)
        affin = memory_affinity(a, req_caps)  # 该 agent 是否有相关记忆(05)，冷启动=None
        scored.append((a, rank(cap, hist, cost, lat, affin)))
    return max(scored, key=lambda x: x[1])[0]

def rank(cap, hist, cost, lat, affin):
    if hist is None or affin is None:          # ★ 冷启动：降权历史/亲和，能力+成本主导
        return 0.6*cap + 0.25*(1-cost) + 0.15*(1-lat)
    return 0.35*cap + 0.25*hist + 0.15*(1-cost) + 0.15*(1-lat) + 0.10*affin
```

### 7.3 能力词表（承重墙，先定）
- 结构：`Domain.Capability`（三层中 Skill 归 04）。受控词表存表 + embedding，**embedding 兜底模糊匹配**。
```sql
CREATE TABLE capability (
  key TEXT PRIMARY KEY,          -- 'procurement.supplier_analysis'
  domain TEXT, name TEXT, description TEXT,
  embedding VECTOR(1024));
CREATE INDEX ON capability USING hnsw (embedding vector_cosine_ops);
```
- Planner 标注、Agent 声明、Router 匹配统一引用 `capability.key`；新增能力走受控流程(避免漂移)。

## 9. ⚠ 实证验证（V1 必做，决定是否进 V2）
用采购分析场景，准备 **10~20 个真实目标**，量化：
| 指标 | 口径 | 阈值(示例) |
|---|---|---|
| DAG 可用率 | 产出 DAG 经校验+人评可执行的比例 | ≥ 70% |
| 路由命中率 | Router 选的 Agent 与人评最优一致 | ≥ 75% |
| 人审通过率 | 端到端产出一次通过 | ≥ 50% |
| 单任务成本/时延 | LiteLLM 统计 | 在预算内 |
未达 → 回炉调 Planner/Router/词表，而非加模块。
