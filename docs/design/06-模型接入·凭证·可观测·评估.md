## 1. 模型网关 = LiteLLM（复用）

### 1.1 ModelGateway 封装
```python
class ModelGateway:
    def resolve(model_spec, org_id) -> ModelHandle:    # 解析模型 + 选档(成本路由)
        candidates = ModelCatalog.by_capability(model_spec.capability, org_id)
        return cost_aware_pick(candidates, Org.budget(org_id))   # 够用选便宜
    async def chat(handle, msgs, tools, cred):         # 经 LiteLLM, 注入用户 Key
        return await litellm.acompletion(model=handle.litellm_name, messages=msgs,
                 tools=tools, api_key=cred.value, metadata={"langfuse": ...})  # ②可观测
    async def embed(text):  return await litellm.aembedding(...)
```
- 统一 OpenAI 协议；LiteLLM 负责多家路由、重试、用量/成本统计。
- 长尾(异步视频/SigV4)：LiteLLM 未覆盖的写薄适配器，仍挂网关后。

## 2. Credential Broker（自研薄封装，底座复用 KMS/Vault）

### 2.1 存储(信封加密)
```sql
CREATE TABLE credential (
  user_id UUID, model_id TEXT,
  ciphertext BYTEA,        -- 用 DEK 加密的用户 Key
  dek_wrapped BYTEA,       -- 用 KMS 主密钥加密的 DEK
  budget_cents BIGINT,
  PRIMARY KEY(user_id, model_id));
```
### 2.2 短时注入(核心安全方法 · 见 IP 08-P3)
```python
class CredentialBroker:
    def scoped(self, org_id, model_id, task_id) -> ScopedCred:
        row = fetch(owner_of(org_id), model_id)
        dek = KMS.unwrap(row.dek_wrapped)               # MVP:应用层KMS; 规模:Vault/云KMS
        key = aes_decrypt(row.ciphertext, dek)
        handle = ephemeral_handle(task_id, key, ttl=task_duration)  # 任务级短时句柄
        audit_log(org_id, model_id, task_id, op="issue")
        return handle                                    # 用完即焚; 永不落盘/日志/上下文/记忆
```
- **多租户隔离命门**：共享 Worker 池里 A 任务只拿 A 的句柄；句柄随任务结束销毁。
- 缺凭证 → 能力缺口，提示用户连模型/选替代（不静默失败）。

## 3. 可观测 = Langfuse（复用）
- 每次 `ModelGateway.chat` / 工具调用 / 节点带 `metadata.langfuse` → 自动上报 Trace。
- Trace 维度：User→Workflow→Agent→Skill→Model；Cost：token/模型/Agent/部门。
- `trace_ref` 表存 Langfuse traceId，关联 plan/task 便于回放。

## 4. ★ Eval（评测方法 · 补的缺口）
```python
class Evaluator:
    def score(node_output, node) -> EvalResult:
        a = assertions(node_output, node.expectedSchema)     # 客观: 格式/字段/无错
        j = llm_judge(node_output, node.acceptanceCriteria)  # 主观: 是否满足验收(LLM-as-judge)
        # 首评刚过阈值时独立复核，取两者低分；明显高/低分保持单评
        if τ <= j.score <= τ + 0.08: j = min(j, independent_judge(node_output, node))
        return EvalResult(passed = a.ok and j.score >= τ, detail=judge_scores)
    def regression(eval_set, version):                       # 改 prompt/Agent 前后对比
        return [score(run(case, version), case) for case in eval_set]
```
- **评测集**：每场景准 10~20 个目标+期望，入库；改动前后跑回归，防"改了变差"。
- **当前落地**：关键节点质量门 `τ=0.6`；首评位于 `[0.6, 0.68]` 才触发独立复核并取低分，避免所有任务双倍成本；固定回归集入口为 `scripts/eval/judge_regression_gate.py`，输出不含样本正文的 JSON 证据。
- 人工抽样打分补主观盲区。Langfuse 承载 judge 结果与对比看板。

## 6. 审批收件箱（统一人审 · 自研粘合）
```sql
CREATE TABLE approval (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID, kind TEXT,        -- plan|dangerous_action|provision_review|skill_review|rework
  ref_id TEXT, payload JSONB,
  status TEXT DEFAULT 'pending', -- pending|approved|rejected
  assignee UUID, decided_by UUID, decided_at TIMESTAMPTZ);
```
```text
计划确认 / 危险动作 / 生成Agent审核 / 返工升级 → 全进 approval 表(一个"待我处理"队列)
用户操作 → 更新 status → 触发 Temporal signal(03) 恢复对应 Workflow
```
