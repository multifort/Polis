# Polis 系统性设计 · 00 总览与设计原则

> 本目录是把前期所有讨论（多智能体平台设计、Polis 七层架构、全链路时序、
> Planner/Capability Routing、以及历次评审与缺口）**整合后的系统性设计**，
> 替代 `docs/architecture/` 下分散的讨论稿。每篇只讲清一个具体问题。
> 版本：V1.0 / 2026-06-16。

## 0. 关键工程决策（已确认 2026-06-16）

1. **完全独立的新系统**：Polis 与现有 hermes-test（Express+SQLite+web+mobile）**不耦合、不迁移、不复用其后端**。
   现有聊天应用保持独立存在；Polis 全新搭建（无历史包袱）。
2. **后端语言 = Python 3.12 + FastAPI**（原生贴 LiteLLM/LangGraph/Guardrails/MCP 等 AI 生态）。
3. **交互层 = 新建 Web 前端**（因不复用现有前端）：建议 React/Next.js，承载
   聊天 + Org 管理 + 计划审批 + 公司看板 + 审批收件箱；前后端 API 解耦，前端语言独立。
   > ⚠ 这是"独立+不复用"带来的新增范围——交互层要从零建（见 `01`/`07` 已纳入）。

## 1. 这套设计要解决什么

把"用户目标"交给一个由多个 AI 角色组成的"虚拟公司"，自动**规划 → 编排 → 执行 → 审计**地完成，
并保证：可控、可审计、数据准确、成本透明、可演进。

## 2. 三条不可动摇的设计原则（贯穿全集）

1. **低运维优先**：组件数量、技术栈、部署复杂度都要压到最低。MVP 单体 + 4 个基础设施，
   不上 Kafka/ES/Milvus/12 微服务；K8s、微服务化是规模到了之后的事（见 `07`）。
2. **开源复用优先，初期不自研**：能用成熟开源组件就直接用（Temporal / LiteLLM / MCP /
   pgvector / Langfuse …，见 `01`）。**只自研"差异化的粘合逻辑"**（编配、能力路由、组织治理、
   出处与信封纪律），不重造基础设施。
3. **渐进式、核心先行**：先用一个真实场景把"大脑"（规划+路由）原型验证出来，用数据决定后续，
   不在验证前过度铺开（见 `07` 路线图、`03` 的实证说明）。

## 3. 整合后的分层架构（已收敛 + 低运维版）

```text
┌─ 交互层        Chat / API（复用现有 web+移动端）
├─ 组织层        Org(虚拟公司) · Agent · Role · Scenario Preset · 角色编配器      [02]
├─ 规划层        Planner(模板优先+LLM) → Plan校验 → 能力路由(节点→Agent)          [03]
├─ 编排层        Temporal（DAG 执行 / 重试 / 人审暂停 / 有界重规划）               [03]
├─ 执行层        Agent Runtime(无状态优先) · Skill(手册+MCP工具) · Guardrails      [04]
├─ 记忆层        统一记忆(任务/角色/组织作用域) + pgvector RAG + 出处              [05]
├─ 模型层        LiteLLM 网关(BYO-Key/成本路由) + Credential Broker               [06]
├─ 可观测层      Langfuse(Trace/Eval/Cost) · 审批收件箱 · Run Manifest             [07]
└─ 数据/部署     PostgreSQL(+pgvector) · 对象存储 · docker-compose→K8s             [07]
```

## 4. 核心 vs 增强（控制初期范围）

| 最小核心（V1 必须） | 增强（明确后置） |
|---|---|
| 薄 Org + 2~3 Agent | 治理/预算/多租户/Marketplace |
| Agent Runtime(无状态) + MCP 工具 | 有状态/长生命周期 Agent、Agent 间通信 |
| Planner(模板优先) + Plan 校验 | LLM 全自动拆解、动态重规划、SOP 强 schema |
| Temporal + 任务记忆 | 三层记忆 + 衰减治理 |
| LiteLLM 模型网关 | 自研 Model Router、媒体/多模态 |
| Langfuse(trace+人工 eval) | 自动 eval/guardrails 全栈 |
