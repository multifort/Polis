# 02 组织、角色与编配 — 详细设计

> **本篇目标**：可直接指导开发的组织/角色/编配详细设计（数据模型 DDL、配置 schema、
> 编配器算法、API、时序、边界），并在末尾给出**任务分解(WBS)与验收标准**。
> 依赖：PostgreSQL、LiteLLM(意图理解/embedding)、`05`记忆、`03`能力词表、`06`审批收件箱。

---

## 1. 范围与职责

本模块负责：组织(Org)与成员管理、Role/Agent 定义与版本、场景预设管理、**角色编配器**
（意图→实例化一个可工作的 Org）。不负责：任务规划(`03`)、执行(`04`)。

## 2. 领域模型与关系

```text
User 1─* OrgMember *─1 Org 1─* Role 1─* Agent 1─* AgentVersion
                         Org 1─* OrgEnabledSkill
                         Org 1─1 OrgMemory(命名空间, 见 05)
ScenarioPreset 1─* PresetAgentTemplate
Agent *─* Capability(通过 agent_capability)
```

## 3. 数据库 DDL（PostgreSQL）

```sql
CREATE TABLE org (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name          TEXT NOT NULL,
  owner_user_id UUID NOT NULL REFERENCES app_user(id),
  charter       TEXT,
  status        TEXT NOT NULL DEFAULT 'active',   -- active|archived
  budget_cents  BIGINT DEFAULT 0,                 -- 组织级预算(分)，见 06
  shared_schema JSONB DEFAULT '{}',               -- 组织内共享术语/产物schema，见 04/05
  policies      JSONB DEFAULT '{}',               -- 权限/审批/数据隔离策略
  created_at    TIMESTAMPTZ DEFAULT now(),
  updated_at    TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE org_member (
  org_id   UUID REFERENCES org(id) ON DELETE CASCADE,
  user_id  UUID REFERENCES app_user(id),
  role     TEXT NOT NULL,         -- owner|approver|member
  PRIMARY KEY (org_id, user_id)
);

CREATE TABLE role (                -- 业务角色(岗位)，Agent≠Role
  id       UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id   UUID REFERENCES org(id) ON DELETE CASCADE,
  name     TEXT NOT NULL,         -- 例:"采购经理"
  description TEXT
);

CREATE TABLE agent (
  id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id    UUID REFERENCES org(id) ON DELETE CASCADE,
  role_id   UUID REFERENCES role(id),
  name      TEXT NOT NULL,
  source    TEXT NOT NULL,        -- preset|generated|custom
  status    TEXT NOT NULL DEFAULT 'draft', -- draft|active|suspended|archived
  current_version TEXT,           -- 指向 agent_version.version
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(org_id, name)
);

CREATE TABLE agent_version (       -- 声明式配置，版本化(可回滚/灰度/AB)
  id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  agent_id  UUID REFERENCES agent(id) ON DELETE CASCADE,
  version   TEXT NOT NULL,         -- semver
  config    JSONB NOT NULL,        -- 见 §4 Agent 配置 schema
  status    TEXT NOT NULL DEFAULT 'draft', -- draft|published|deprecated
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(agent_id, version)
);

CREATE TABLE agent_capability (    -- 用于能力路由(03)
  agent_id   UUID REFERENCES agent(id) ON DELETE CASCADE,
  capability TEXT NOT NULL,        -- 受控词表项, Domain.Capability
  level      TEXT,                 -- basic|advanced
  PRIMARY KEY (agent_id, capability)
);

CREATE TABLE scenario_preset (
  id      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name    TEXT NOT NULL,
  version TEXT NOT NULL,
  description TEXT,
  required_capabilities TEXT[],
  config  JSONB NOT NULL,          -- agentTemplates[] + defaultPlanTemplates[]
  embedding VECTOR(1024),          -- 预设语义检索(pgvector)，按 description+caps 生成
  UNIQUE(name, version)
);
CREATE INDEX ON scenario_preset USING hnsw (embedding vector_cosine_ops);
```

## 4. 配置 schema（声明式，入 `agent_version.config`）

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| prompt | string | 是 | 系统提示词 |
| model | {provider, model, params} | 是 | 经 LiteLLM 解析(06) |
| skills | string[] | 否 | `name@version`，手册型或工具型(04) |
| capabilities | string[] | 是 | 受控词表能力(03)，路由用 |
| executor | enum | 是 | lite-agent\|pure-llm\|media |
| authority | {allowedTools[], dangerousActions} | 是 | 最小权限(04/06) |
| memory | {namespace, scopes[]} | 否 | 记忆作用域(05) |
| policies | {max_tokens, max_tool_calls, data_access} | 是 | 资源/数据策略 |

Preset.config：
```jsonc
{ "agentTemplates": [
    { "roleName":"采购经理", "agentName":"询价Agent",
      "promptSkeleton":"...", "skills":["erp-query"], "capabilities":["procurement.supplier_analysis"] }],
  "defaultPlanTemplates": ["supplier_analysis_v1"] }   // 引用 03 的 Plan 模板
```

## 5. 角色编配器（自研核心 · 详细设计）

### 5.1 子模块
```text
Provisioner
 ├─ IntentParser       onboarding 问答 → 结构化意图
 ├─ PresetMatcher      意图 → 检索最匹配 ScenarioPreset(pgvector)
 ├─ Assembler          预设 agentTemplates → 实例化 Agent(draft)
 ├─ GapGenerator       缺口能力 → LLM 生成 Agent 配置(draft, source=generated)
 ├─ AgentValidator     生成的 Agent 跑样例 + 断言 + LLM-judge(见 06)
 └─ OrgInstantiator    建 Org + 填充 Agent + 启用技能 + 建组织记忆命名空间
```

### 5.2 主流程伪代码
```python
def provision(user, answers) -> Org:
    intent = IntentParser.parse(answers)          # LLM → {domain, goalType, prefs, requiredCaps[]}
    preset, score = PresetMatcher.best(intent)    # embedding 检索 scenario_preset
    org = OrgInstantiator.create(user, intent)
    covered = set()
    if preset and score >= MATCH_THRESHOLD:        # 命中/部分命中
        for tpl in preset.agentTemplates:
            ag = Assembler.instantiate(org, tpl, intent.prefs)  # draft
            covered |= set(ag.capabilities)
    gaps = set(intent.requiredCaps) - covered      # 缺口能力
    for cap in gaps:                                # 缺口→生成(需人审)
        spec = build_agent_spec(cap, intent)
        ag = GapGenerator.generate(org, spec)       # draft, source=generated
        if not AgentValidator.check(ag):            # 跑样例+断言+judge
            mark_needs_human(ag)
    Approval.submit(org, kind="provision_review")   # 统一审批收件箱(06)：预览+人审
    return org                                       # 人审通过后 agent.status→active
```

### 5.3 关键判定
- `MATCH_THRESHOLD`：余弦相似度阈值(初值 0.75，可调)；低于则走"不命中→全生成"。
- **生成 Agent 默认 `status=draft`**，人审通过才 `active`；意图误读靠人审兜底。
- 冷启动：预设库空时全部走 GapGenerator；预设随用随沉淀(把验收通过的编配存回 preset)。

## 6. API（REST，挂现有 server）

```http
POST /api/orgs                      # 直接建空 Org
POST /api/provision                 # body:{answers} → 启动编配 → 返回 orgId + 待审 agents
GET  /api/orgs/{id}                 # Org 详情(花名册/技能/预算)
POST /api/orgs/{id}/agents          # 手动加 Agent
GET  /api/agents/{id}/versions      # 版本列表
POST /api/agents/{id}/publish       # 发布版本(draft→published, agent→active)
POST /api/agents/{id}/rollback      # body:{version}
```

## 7. 编配时序

```text
User → POST /provision → IntentParser(LLM) → PresetMatcher(pgvector)
 → [命中] Assembler 实例化 Agents(draft)
 → [缺口] GapGenerator(LLM) → AgentValidator(样例+judge)
 → OrgInstantiator 建 Org+记忆命名空间
 → Approval 收件箱(预览+人审) → 通过 → agents.active → 返回 Org
```

## 8. 边界与错误处理

- 意图无法解析 → 回退到"让用户从预设列表手选"。
- 缺口能力无对应 skill/model → 记为待办(不阻塞建 Org)，提示用户去连/补。
- 生成 Agent 验证失败 → 保持 draft + needsHuman，不进花名册执行。
- 并发：同名 Agent 唯一约束 `UNIQUE(org_id, name)`，冲突自动追加序号。

## 9. 任务分解(WBS) 与验收

| 任务 | 交付 | 验收标准 |
|---|---|---|
| T2.1 DDL + 迁移 | org/role/agent/version/capability/preset 表 | 迁移可执行，外键/索引生效 |
| T2.2 Agent/Preset 配置 schema + 校验 | JSON Schema 校验器 | 非法配置被拒并给出字段级报错 |
| T2.3 IntentParser | LLM 提示词 + 解析器 | 10 个样例意图解析准确率人评 ≥ 80% |
| T2.4 PresetMatcher | pgvector 检索 | 命中 Top1 在人评相关集内 ≥ 80% |
| T2.5 Assembler + OrgInstantiator | 建 Org + 实例化 Agent | 预设场景一键建出可用花名册 |
| T2.6 GapGenerator + AgentValidator | 生成+验证闭环 | 生成 Agent 默认 draft，验证不过不上线 |
| T2.7 Provision/Org/Agent API | REST | 端到端 provision 通 + 人审后 active |
| T2.8 编配端到端联调 | 采购场景 | 从问答到建出"采购公司" < 2 分钟、可进 03 规划 |
