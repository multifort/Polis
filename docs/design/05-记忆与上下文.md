# 05 记忆与上下文 — 详细设计

> **本篇目标**：可指导开发的统一记忆系统详细设计——DDL(pgvector)、写入/检索/衰减管线、
> 作用域权限、结果信封、共享并发一致性、API + WBS。
> 依赖：PostgreSQL+pgvector、LiteLLM(embedding/rerank)、`04`执行、`03`/`06`。

---

## 1. 模型：一套存储，作用域+类型区分线路

```text
scope: session | task | role | org          # 线路区分(个体/共享)
type:  factual | procedural | preference | event
```
- session：单会话(弱持久)；task：单任务台账；role：某 Agent 跨任务积累；org：组织共享。

## 2. 数据库 DDL（PostgreSQL + pgvector）

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE memory (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id      UUID NOT NULL,
  scope       TEXT NOT NULL,            -- session|task|role|org
  namespace   TEXT NOT NULL,            -- 如 'procurement' / agentId / taskId
  type        TEXT NOT NULL,            -- factual|procedural|preference|event
  content     TEXT NOT NULL,
  embedding   VECTOR(1024),
  provenance  JSONB,                    -- {sourceUrl, retrievedAt, confidence}
  importance  REAL DEFAULT 0.5,
  confidence  REAL DEFAULT 0.5,
  decay_rate  REAL DEFAULT 0.01,
  created_at  TIMESTAMPTZ DEFAULT now(),
  last_accessed TIMESTAMPTZ DEFAULT now(),
  expires_at  TIMESTAMPTZ               -- session/task 可设 TTL
);
CREATE INDEX ON memory USING hnsw (embedding vector_cosine_ops);
CREATE INDEX ON memory (org_id, scope, namespace);

CREATE TABLE result_envelope (          -- 任务台账(有界)
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID, node_id TEXT, agent_id UUID, status TEXT,
  artifacts JSONB,                      -- [{uri, kind}]
  facts JSONB,                          -- [{claim, sourceUrl, retrievedAt, confidence}]
  summary TEXT, needs_human BOOL DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now());

CREATE TABLE artifact_descriptor (      -- 多模态产物描述符(字节在对象存储)
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID, node_id TEXT,
  modality TEXT, uri TEXT, mime TEXT, caption TEXT,
  provenance JSONB, meta JSONB,
  created_at TIMESTAMPTZ DEFAULT now());
```

## 3. 写入管线
```python
def write(org_id, agent, envelope):
    for fact in envelope.facts:
        ext = Extractor.normalize(fact)                  # 抽取标准化
        score = ext.importance * fact.confidence         # 评分
        if Filter.is_noise_or_dup(ext, org_id): continue # 过滤去噪/去重
        emb = ModelGateway.embed(ext.content)            # LiteLLM embedding
        upsert_memory(org_id, scope=agent.memory.scope, namespace=agent.memory.namespace,
                      type=ext.type, content=ext.content, embedding=emb,
                      provenance=fact.provenance, importance=score, confidence=fact.confidence)
    save_result_envelope(envelope)                        # 进台账
```

## 4. 检索管线（RAG · 解决"怎么取"）
```python
def retrieve(org_id, agent, node) -> Slice:
    q = node.inputHint + " " + node.requiredCapabilities
    emb = ModelGateway.embed(q)
    rows = sql("""
      SELECT id, content, provenance, importance, last_accessed
      FROM memory
      WHERE org_id=%s AND scope = ANY(%s)            -- ★作用域权限过滤
        AND namespace = ANY(%s)
      ORDER BY embedding <=> %s LIMIT 40
    """, org_id, agent.memory.scopes, allowed_namespaces(agent), emb)
    ranked = rerank(rows, by=[relevance, recency, importance])  # reranker(LiteLLM)
    touch_last_accessed(ranked)
    return Slice(summaries=[r.content for r in ranked[:K]],     # ★注入摘要不是全量
                 provenance=[r.provenance for r in ranked[:K]])
```
- `allowed_namespaces`/`scopes` 由 Agent 配置 + Org 策略决定（治理）。

## 5. 衰减/遗忘（定时任务）
```python
def decay_job():     # 每日
    sql("UPDATE memory SET importance = importance * exp(-decay_rate * age_days)")
    sql("DELETE FROM memory WHERE importance < 0.05 AND type='event'")  # 低价值清理
    sql("DELETE FROM memory WHERE expires_at < now()")                  # session/task TTL
```
衰减率：factual/preference 慢、event 中、噪声快。

## 6. ★ 共享记忆并发一致性（补的缺口）
并行分支同时写 org 作用域：
```python
def upsert_shared_fact(org_id, fact):
    old = find_similar(org_id, scope='org', fact)        # 语义近邻
    if old is None: insert(fact)
    elif fact.confidence > old.confidence: override(old, fact)   # 高置信覆盖
    else: mark_conflict(old, fact)                       # 冲突→标记，交审核角色/人裁决
```
- 读侧：检索带 confidence；审核角色/人审处理 `conflict` 标记，不静默硬合并。

## 7. ★ 数据准确性（出处贯穿全链路）
`工具产出(04) → 结果信封.facts(带 sourceUrl/confidence) → 写入 memory.provenance → 检索带出处 → 审核角色核对引用/时效`。

## 8. API
```http
POST /api/memory/write       # 内部: 由 AgentRuntime 调
POST /api/memory/retrieve    # body:{orgId, query, scopes, namespaces} → slice
GET  /api/orgs/{id}/memory   # 浏览/治理(审计)
DELETE /api/memory/{id}      # 治理删除(权限校验)
```

## 9. 治理
- 权限：Agent 声明可读 scope/namespace；写仅限 own + 显式授权的 org。
- 审计：memory 读写记日志(谁/何时/哪条)。
- 安全：敏感字段脱敏/加密；Org 间 `org_id` 强隔离(行级)。

## 10. 复用与自研
- 复用：pgvector(检索)、embedding/reranker(LiteLLM)、可借鉴 Mem0/Letta 模式。
- 自研：出处纪律、结果信封、作用域受限检索、共享并发裁决。

## 11. 任务分解(WBS)
| 任务 | 交付 | 验收 |
|---|---|---|
| T5.1 memory/envelope/artifact DDL + pgvector | 迁移 | hnsw 索引生效，向量检索可用 |
| T5.2 写入管线(抽取/评分/过滤) | write() | 噪声/重复被过滤，出处入库 |
| T5.3 检索管线(RAG+rerank+作用域过滤) | retrieve() | 跨作用域权限正确，返回摘要 |
| T5.4 衰减/遗忘定时任务 | decay_job | 低价值/过期被清理 |
| T5.5 共享并发裁决 | upsert_shared_fact | 高置信覆盖、冲突标记 |
| T5.6 治理(权限/审计/隔离) | 行级隔离+审计日志 | A 不能读 B 的 org 记忆 |
| T5.7 与 04 联调 | 执行→写回→再检索 | 第二个 Agent 能读到第一个写的带出处事实 |
