# 08 关键技术与知识产权（候选专利点）

> **解决的问题**：从整套设计里提炼**可申请知识产权**的关键技术方法。
> ⚠ 我不是专利代理人。以下是**候选技术点 + 诚实的新颖性评估**，供你与专业代理人评估、检索在先技术后定稿。
> 通用件（RAG、Temporal 编排、向量检索本身）是公知技术，**不具可专利性**；
> 真正可主张的是下面这些**具体方法 / 独特组合**。

---

## 候选点一览（按可主张性从强到弱）

### ★★ 强（独特方法/组合，建议优先评估）

**P1. 意图驱动的多智能体"虚拟公司"自动编配方法**
- 内容：由用户 onboarding 意图 →（匹配场景预设 + 组装预设角色 + 对能力缺口自动生成角色 + 验证/人审）
  → 自动实例化一个含花名册/共享资源/治理的"组织容器(Org)"。
- 可主张点：把"意图理解→预设组装→缺口生成→组织实例化"做成一条**自动编配流水线**。
- 新颖性：多 agent 框架普遍要手工配角色；"按意图自动编出一个可治理组织"较少见。（`02`）

**P2. 带记忆亲和度与冷启动回退的能力路由方法**
- 内容：节点能力需求 → 候选 Agent 加权打分 `0.35 能力匹配 + 0.25 历史成功 + 0.15 成本 + 0.15 延迟 + 0.10 记忆亲和`；
  **冷启动时自动降权历史/亲和项、回退到能力+成本主导**，随运行积累再启用完整公式。
- 可主张点：**记忆亲和度作为路由因子** + **冷启动自适应权重**的组合。
- 新颖性：能力路由有先例，但"记忆亲和 + 冷启动自适应"的具体组合较独特。（`03`）

**P3. 面向 BYO-Key 多租户的"无状态共享 Worker + 任务级短时凭证注入"方法**
- 内容：通用无状态 Worker 池；执行某用户任务时，将其 BYO 凭证经信封加密解出、以**任务级短时句柄**注入，
  用完即焚，永不落盘/日志/上下文；实现共享池下的跨租户密钥隔离。
- 可主张点：**短时凭证注入 + 即焚 + 跨租户隔离**用于多 agent 共享执行池的具体机制。
- 新颖性：秘密注入有通用方案，但"无状态多 agent 池 + 任务级即焚 + BYO 多租户"的组合可主张。（`06`）

**P4. 多智能体数据准确性保障方法（出处结果信封 + 作用域受限记忆）**
- 内容：每个 agent 产出**带出处+置信度的结果信封**进有界台账；事实写入共享记忆时携带来源；
  作用域受限的 RAG 检索 + 并发冲突按置信度裁决/标记；审核角色据出处核对引用与时效。
- 可主张点：**出处贯穿"信封→记忆→检索→审核"全链路**以对抗多 agent 误差累积与互联网数据不可靠。
- 新颖性：出处/溯源有先例，但"贯穿多 agent 协作全链路的数据准确性保障方法"可主张。（`05`）

### ★ 中（方法清晰，但更接近已知思路，需检索在先技术）

**P5. 自扩展技能闭环（缺口检测→规格→检索/生成→验证沙箱→信任分级→人审升档）**
- 内容：检测能力缺口→生成技能需求规格→检索或 LLM 生成 SKILL.md→受限沙箱断言+LLM-judge 验证→
  默认最低信任档→人审升档入库。
- 可主张点：**"自动生成 + 验证沙箱 + 信任分级 + 人审升档"闭环**保证自扩展安全。（`02`/相关讨论）

**P6. 双形态技能的运行时无关执行方法**
- 内容：技能分**手册型(Markdown playbook，讲怎么做)** 与 **工具型(MCP，讲用什么做)**，二者解耦；
  同一手册可注入任意执行器运行，工具统一经 MCP 调用。
- 可主张点：手册/工具解耦 + 运行时无关 的统一执行。（`04`）

**P7. LLM 生成计划的"校验 + 有界动态重规划"方法**
- 内容：对 LLM 产出的 DAG 做**自动校验**(无环/完整/预算/能力可满足/危险动作已标人审)拦截垃圾计划；
  执行中节点失败/不达标触发**有界局部重规划**(非整盘、非无限自主)。
- 可主张点：**计划校验门 + 有界重规划**的组合。（`03`）

### ☆ 弱（多为公知组合，作为辅助或不单独申报）

- **P8. 三作用域统一记忆 + 衰减治理**：作用域(任务/角色/组织)统一存储 + 指数衰减 + 治理。记忆衰减/分层有较多在先技术，建议作为 P4 的从属特征，不单独主张。

---

## 申报建议

1. **优先 P1～P4**（方法独特性与组合新颖性较强），各写成独立**方法专利**（步骤+系统两套权利要求）。
2. P5～P7 作为**从属或防御性**申报，或并入 P1/P3 的实施例。
3. **务必先做在先技术检索**（CrewAI/AutoGen/LangGraph/MetaGPT/DeskClaw 及相关专利），由代理人核新颖性与创造性。
4. 申报材料可直接取本设计集 `02~06` 的流程图/数据模型作为**说明书附图与实施例**。

> 再次声明：以上为工程视角的候选点与诚实评估，**最终可专利性以专业检索与代理人意见为准**。
