## 3. 多租户隔离与 RLS（落实 09§5 / ADR-0005）

- **组织级表**：均带 `org_id`，启用 RLS（FORCE）。策略：
  `USING (org_id = NULLIF(current_setting('app.current_org', true), '')::uuid)`。
  用 `NULLIF` 把空串归 NULL——`app.current_org` 被 set 后 `RESET` 会变 `""`，否则 `''::uuid` 会报错；
  空串/未设均 **fail-closed 返回 0 行**。
- **平台级表**(app_user/auth_session)：不启用 RLS。
- **全局共享表**(capability/model_catalog/preset/plan_template/skill/skill_version)：不带 org_id、不启用 RLS（只读为主，写经受控流程）。
- **用户级表**(credential)：按 `user_id` 隔离（应用层），不套 org-RLS。
- **org_invite 特殊**：带 org_id 但邀请对象尚非成员——按 token 访问，不走 current_org RLS（应用层校验 token）。

### 3.1 强制机制：`SET ROLE polis_app`（无密码、无入库密钥）
- 迁移创建 **`polis_app`（NOLOGIN、非 superuser）** 角色 + 授予 CRUD 授权（见迁移 `3ffe8c94852b`）。
- 业务连接以现有账号登录后，**每请求** `SET ROLE polis_app` + `set_config('app.current_org', <org>, true)`，
  使该会话受 RLS 强制（非表 owner、非 superuser → RLS 生效）。请求结束 `RESET ROLE`。
  这样无需为 app 单独建带密码的登录角色，**零密钥入库**。
- 双保险：应用层 repo 仍统一注入 `org_id` 过滤（第一道）；RLS 是第二道兜底。运维/迁移用 superuser（绕过 RLS）。
- **隔离回归 T8.3**：`tests/test_integration_rls.py` 已验证 A/B 两租户互不可见 + 未设上下文 fail-closed。
- ⚠ 应用按请求 `SET ROLE`+`current_org` 的中间件接线随 **M2 当前公司中间件(T9.2)** 落地；本批已备好角色/策略并测通机制。

## 4. 命名与通用约定（落实 12 C 系列）

- 主键 `id UUID DEFAULT gen_random_uuid()`；表/列 `snake_case`；时间 `TIMESTAMPTZ`。
- 枚举一律 **`TEXT` + `CHECK`**（或应用层 Pydantic 校验），**不用 PG ENUM**（ENUM 难演进）。
- 软扩展字段用 **`JSONB`**（`config`/`policies`/`meta`/`provenance`/`payload`…）。
- 向量列 `VECTOR(1024)` + `hnsw (… vector_cosine_ops)` 索引。
- 每个组织级表带 `org_id`（FK→org）；删除策略多为 `ON DELETE CASCADE`（org 删则其下级联）。

## 5. 扩展策略（加表/加字段不返工）

1. **迁移向后兼容**(14 E9)：只走 Alembic；先加列/表→灰度→再删；每迁移可 `downgrade`；与代码同 PR。
2. **JSONB 软扩展**：可选/高频变动属性进 JSONB，不为加字段动 schema。
3. **版本化资源**：`agent_version`/`skill_version`/`plan_template` 演进行为，不改既有表。
4. **受控词表**：`capability` 经受控新增流程扩，不改 schema、不漂移。
5. **状态用 TEXT+CHECK**：新增枚举值=改 CHECK（或纯应用层），不像 ENUM 要 `ALTER TYPE`。
6. **模块自有模型**：新表只在所属 `modules/<x>/models.py` 加，`db/models.py` 汇总；模块边界清晰。
7. **向量维度变更**：不原地改维度——加新列/新表 + 重建 hnsw + 双写灰度迁移。
8. **新表默认带 org_id + RLS**（除非明确属平台/全局/用户级），并补隔离回归(T8.3)。

## 6. 迁移组织
- 全量 DDL 首迁移由 SQLAlchemy 模型 `autogenerate` 生成（人审后落库）。
- RLS 启用 + 策略、`vector` 扩展、hnsw 索引、CHECK 约束等 autogenerate 覆盖不到的，**在迁移中手工补**（`op.execute`）。
- seed（capability/model_catalog/preset）用独立幂等脚本（`make seed`），不混进结构迁移。
