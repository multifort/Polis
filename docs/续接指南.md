## 4. 技术栈与关键架构
- 后端：Python 3.12 + FastAPI + SQLAlchemy 2.0(async) + Alembic + Pydantic v2，`uv` 管理。前端：Next.js 14 App Router + TS + pnpm。
- 分层（照 `modules/org` 写）：`schemas.py`(Pydantic) / `repository.py`(集中 SQL) / `service.py`(业务，抛领域异常) / `api.py`(路由，异常→HTTP)。模块在 `backend/src/polis/modules/{org,planner,runtime,memory,model,observability}`。
- **多租户**：组织级端点用依赖 `from polis.modules.org.deps import CurrentOrg`（校验 `X-Org-Id` 成员 + 每请求 `SET LOCAL ROLE polis_app` + `set_config('app.current_org',…)` → RLS 生效）。`CurrentUserId` 取当前用户。`get_session` 是**每请求一个事务**，service 只 `flush` 不 commit。RLS 细节见 `docs/design/0b §3`、`docs/decisions/0005`。
- 数据模型权威地图：`docs/design/0b`（27 表 + 隔离分层 + 扩展策略）。迁移只走 Alembic、向后兼容。

## 6. ⚠ 环境踩坑（避免重复踩）
1. **会话开在 `/Users/lining/workspace/polis`**。若开在别的项目根（如某 hermes 仓库），cwd/git 会串台。
2. **后台子代理（run_in_background / 无人值守）没有 Bash/Read 权限**，且 worktree 会基于会话根仓库而非 Polis → 跑不通。要委派就用**前台子代理**（一次一个、会阻塞、但有 Bash），或直接自己做。
3. Bash 工具：涉及网络/docker/uv/pytest起容器/git push 的命令要传 `dangerouslyDisableSandbox: true`；shell 有 `set -e`，用 `grep` 前加 `set +e;`，复杂命令易吞输出，必要时写 `/tmp` 再 Read。
4. **pre-commit 钩子**会用 ruff-format 改文件导致首次 `git commit` 中止——遇到就 `git add -A` 再 commit 一次。
5. **后端改了代码要用 `make dev`（带 --reload）**；手动 `nohup uvicorn …`（无 --reload）会一直跑旧代码，调试会被坑（已踩两次）。
6. mypy strict：`session.scalar(...)` 返回 Any → 先赋给带类型标注的变量再 return；ORM 的 `Mapped[dict]` 要写 `Mapped[dict[str, Any]]`；第三方缺 stub 在 `pyproject.toml` 加 `[[tool.mypy.overrides]] module=["x.*"] ignore_missing_imports=true`。
7. pgvector 用 pg18 镜像（华为云源拉取后 retag）；pg18 卷挂 `/var/lib/postgresql`（非 `/data`）。
