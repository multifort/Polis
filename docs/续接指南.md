## 6. ⚠ 环境踩坑（避免重复踩）
1. **会话开在 `/Users/lining/workspace/polis`**。若开在别的项目根（如某 hermes 仓库），cwd/git 会串台。
2. **后台子代理（run_in_background / 无人值守）没有 Bash/Read 权限**，且 worktree 会基于会话根仓库而非 Polis → 跑不通。要委派就用**前台子代理**（一次一个、会阻塞、但有 Bash），或直接自己做。
3. Bash 工具：涉及网络/docker/uv/pytest起容器/git push 的命令要传 `dangerouslyDisableSandbox: true`；shell 有 `set -e`，用 `grep` 前加 `set +e;`，复杂命令易吞输出，必要时写 `/tmp` 再 Read。
4. **pre-commit 钩子**会用 ruff-format 改文件导致首次 `git commit` 中止——遇到就 `git add -A` 再 commit 一次。
5. **后端改了代码要用 `make dev`（带 --reload）**；手动 `nohup uvicorn …`（无 --reload）会一直跑旧代码，调试会被坑（已踩两次）。
6. mypy strict：`session.scalar(...)` 返回 Any → 先赋给带类型标注的变量再 return；ORM 的 `Mapped[dict]` 要写 `Mapped[dict[str, Any]]`；第三方缺 stub 在 `pyproject.toml` 加 `[[tool.mypy.overrides]] module=["x.*"] ignore_missing_imports=true`。
7. pgvector 用 pg18 镜像（华为云源拉取后 retag）；pg18 卷挂 `/var/lib/postgresql`（非 `/data`）。

## 7. M3 下一步（C、D 的详细任务）

### M3-C：Temporal 编排 + 有界重规划（桩执行器，无 LLM）—— 在 `feat/m3-planner` 主工作树
1. `cd backend && uv add temporalio`。`config.py` 加 `temporal_addr: str = "localhost:7233"`。Makefile 加 `worker:` 目标。
2. `modules/planner/workflow.py`：
   - `@activity.defn run_node(node: dict, org_id: str) -> dict`：桩执行器（按 node 调 `router.select_agent` 选 Agent，返回 `{node_id, ok, agent, output:"[stub]…"}`；node 含 `fail_once` 时首次抛错验证 retry）。
   - `@workflow.defn TaskWorkflow`：入参 `plan: dict`(PlanDag.model_dump()) + `org_id`；按依赖**拓扑+并行**执行（asyncio.gather 并发 execute_activity，`RetryPolicy(maximum_attempts=3)`）；`type=="human"` 节点 `wait_condition` 挂起，`@workflow.signal approve(node_id)` 恢复；`@workflow.query status()` 返回各节点状态。
   - 有界重规划：节点 soft-fail → 局部重排（≤K 次、patch 仅替换/重排个别节点、重排后过 `validate`），超 `MAX_REPLANS` → 整体 failed。
3. `modules/planner/worker.py`：连 `TEMPORAL_ADDR`，注册 TaskWorkflow + run_node，task_queue `polis-tasks`。
4. API（挂 `planner/api.py`，org-scoped 用 CurrentOrg）—— **契约（D 已按此写前端）**：
   - `POST /api/plans/{plan_id}/approve` → 建 `task_run`，Temporal Client `start_workflow(TaskWorkflow,[plan.dag,str(org_id)],id=f"plan-{plan_id}",task_queue="polis-tasks")`，回 `{task_id,status:"running"}`。
   - `GET /api/plans/{plan_id}/run` → `handle.query(TaskWorkflow.status)`，回 `{status, nodes:[{id,status,agent}]}`；Temporal 不可达回 **503**（不是 500）。
   - `POST /api/plans/{plan_id}/signal` body `{node_id}` → `handle.signal(approve,node_id)`，回 204。
5. 测试 `tests/test_workflow.py`：用 `temporalio.testing.WorkflowEnvironment.start_time_skipping()`（不依赖 docker temporal，首次下载测试 server 二进制，需联网）跑端到端：串/并全 done、human 挂起→signal 完成、fail_once 经 retry 成功、重规划上限。
6. （best-effort）起 compose temporal：`cd infra && docker compose --env-file .env up -d temporal temporal-ui`（拉镜像可能慢/失败，失败就跳过，API 已能 503 降级）。
