# ADR-0001：Polis 与 hermes-test 完全独立

- 状态：accepted
- 日期：2026-06-16

## 背景
Polis（多 Agent 协同平台）与既有 `hermes-test`（Expo/RN 聊天客户端 + Node 服务）在领域、技术栈、演进节奏上差异很大。
需决定是在 hermes-test 内扩展，还是另起独立系统。

## 选项
1. 在 hermes-test 内扩展 —— 复用其前端/服务，但耦合既有数据模型与发布节奏，约束多、技术栈被绑定。
2. 完全独立的新系统 —— 自由选型，边界清晰；代价是不复用既有前端、需新建基础设施。

## 决定
**完全独立**：新建 `polis` 仓库与代码库，不复用 hermes-test 前端；仅迁移设计与规范文档。

## 后果
- 正面：技术选型自由（Python 后端、独立部署），边界清晰，互不拖累。
- 负面 / 代价：前端需重建（React/Next.js）；两套各自维护。
- 影响范围：仓库、CI、部署、团队约定均按 Polis 独立建立。
