# ADR-0010：角色驱动为主干 + 任务驱动弹性（混合编配）

- 状态：proposed
- 日期：2026-06-22
- 关联：[V2 · 生成式内核 §5](../design/v2/01-生成式内核（编配器·规划器·能力模型·信任）.md)

## 背景
建公司后如何组织 Agent 去办任务，两种范式：
- **任务驱动**：给定任务 → 临时按需组队。
- **角色驱动**：先定义公司组织/角色 → 任务路由到已有角色。
V1 现态是角色驱动（预设花名册）。需定 V2 走哪种。

## 选项
1. 纯**任务驱动** —— 灵活、能办新任务；但无稳定组织身份，难积累 org 记忆/路由历史/亲和度，每任务生成成本高。
2. 纯**角色驱动** —— 稳定、可积累、贴"虚拟公司"隐喻；但花名册覆盖不到的任务办不了。
3. **混合**：角色驱动为底座 + 任务驱动为弹性补充。

## 决定
**混合，以角色驱动为主干**：
- 公司有稳定花名册（贴隐喻；稳定组织才能积累 org 记忆、路由历史、亲和度——V2 路由/记忆的燃料）；
- 任务需要现有花名册覆盖不了的能力时，**按任务临时组队**（拼 Skill 成临时 Agent）；
- **高频复用的临时角色 → 晋升为常设**。
- 与"检索→生成退化链"同构：常设花名册=检索命中；临时组队=生成兜底；高频临时→沉淀常设。

## 后果
- 正面：稳定可积累 + 能办新任务；与内核退化链、记忆/路由历史一致。
- 负面/代价：临时组队与常设晋升的治理（触发阈值，与记忆晋升对齐）。
- 影响范围：编配器、路由、仓库（角色模板沉淀）。
