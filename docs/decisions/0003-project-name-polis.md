# ADR-0003：项目命名为 Polis

- 状态：accepted
- 日期：2026-06-16

## 背景
工作期使用代号 “AgentOS”，但该名在 AI 领域已被占用（如 Letta），辨识度低、易混淆。需定正式名。

## 选项
1. 沿用 AgentOS —— 撞名、品牌风险。
2. Polis（城邦）—— 每个 Org = 一个自治城邦：有公民(Agent)、角色分工、治理(审批/护栏)，扣住「虚拟公司」内核；短、可作包名、不撞车。
3. 其他候选：Cadre / Praxis / Maestro。

## 决定
**Polis**。仓库 `polis`，包/命名空间 `polis`。活动文档已统一为 Polis，`docs/legacy/` 归档保留旧字样作为历史。

### 补充（2026-06-18）：对用户的品牌定位
"Polis（城邦）"直译对用户不够自解释。决定**保留 Polis 字标不变**（代号/仓库/包名零改动），
对用户**统一加中文定位「虚拟智能企业平台」+ 标语"一个目标，开出一家 AI 虚拟公司"**，让人一眼懂系统作用。
- 不改名（不选"智企/AgentCorp"等）：避免 repo/包/ADR 返工，且 Polis 字标已建立。
- 落地：登录页 Hero/卡片、`frontend` metadata、README 标题与 `docs/design/0a` 均以此定位呈现。

## 后果
- 正面：辨识度高、隐喻贴合、命名空间干净。
- 负面 / 代价：已完成活动文档的 “AgentOS→Polis” 术语替换；`docs/legacy/` 保留旧字样作为历史。
- 影响范围：仓库名、包名、对外品牌、文档术语表(A14)。
