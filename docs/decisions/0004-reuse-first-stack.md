# ADR-0004：复用优先的基础技术栈

- 状态：accepted
- 日期：2026-06-16

## 背景
多 Agent 平台涉及编排、模型网关、工具协议、向量检索、可观测/评估、注入防御等多个子系统。
若全自研，工期与运维成本高、风险大。需定 MVP 的基础件策略。

## 选项
1. 自研各子系统 —— 可控但慢、坑多、运维重。
2. 复用成熟开源件直接组装，先不自建 —— 上手快、运维轻；代价是受组件能力边界约束。

## 决定
**复用优先**，MVP 仅引入 4 个基础设施，避免过度铺开：
- 编排：**Temporal**（持久化工作流 / 重试 / 补偿）
- 模型网关：**LiteLLM**（多模型 / BYO-key / 成本计量）
- 工具协议：**MCP**
- 数据 + 向量：**PostgreSQL + pgvector**（低运维，暂不上 Milvus）
- 可观测 / 评估 / 成本：**Langfuse**
- 注入防御：**Guardrails-AI**；运行时用 LangGraph 或最小 loop。
部署 docker-compose → 规模期再 K8s（CI/CD、容器编排为后置项，见 `docs/constraints/14`）。

## 后果
- 正面：减少自研、运维轻、快速验证；把精力留给真正差异化（规划/路由/记忆/凭证）。
- 负面 / 代价：受组件边界约束；未来若某件不够用需评估替换（走新 ADR）。
- 影响范围：infra/、backend 依赖、部署形态、里程碑 M1–M7（见 `docs/design/07`）。
