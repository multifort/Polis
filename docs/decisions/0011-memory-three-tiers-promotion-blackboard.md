# ADR-0011：记忆三层 + 自动晋升 + 任务黑板按需拉取

- 状态：proposed
- 日期：2026-06-22
- 关联：[V2 · 记忆系统](../design/v2/02-记忆系统（三层·晋升·黑板·预算）.md)

## 背景
V1（M5）已有 org/role/task 作用域 + 向量 RAG + 衰减。两个问题：
1. **F3 实证**：下游节点看不到上游产出，"整合前面分析"靠模糊检索碰运气——任务内协同无可靠通路。
2. 若把上游产出**全链路推给下游**，token 成本随 DAG 规模爆炸。

## 选项
1. 节点间**全链路传完整产出** —— 可靠但 token 爆炸。
2. 纯靠**模糊向量检索**（V1 现状）—— 便宜但不可靠（碰运气）。
3. **共享存储 + 按需拉取**：任务黑板 + 按 id 懒加载；跨任务靠晋升 + RAG。
4. 晋升：人工 curate（干预多）/ **自动+阈值**（数据级低风险）。

## 决定
- **三层（四作用域 session/task/role/org）**：org/role 存**蒸馏 fact**，task 存**原始 envelope**。
- **一切按需 pull，从不推全链路**：
  - 任务内 = **黑板**（`result_envelope` + 新增 `content` 全文）+ **按 id 懒加载**（确定性技能 `read_node_output`）+ 限额检索；直接依赖的**摘要自动带上**（可靠兜底）。
  - 任务间 = **自动晋升 + 按需 RAG**。
- **自动晋升、无人 curate**（importance/confidence/频次阈值；**org 级置信门更高 + 必带出处**，防幻觉沉淀为公司知识）。
- 取数=确定性技能、决策=LLM；全程 **token 预算受控**（挂成本门）。

## 后果
- 正面：上下文与 DAG 规模解耦（成本可控）；任务内协同可靠（修 F3 缺陷）；公司知识自增长。
- 负面/代价：晋升阈值需按数据校准；`result_envelope` 加 `content/tokens`；org 并发写靠 `upsert_shared_fact` 裁决。
- 影响范围：记忆中心、执行运行时（黑板/工具）、协同框架（重跑取上游态）、内核（org 记忆接地）。
