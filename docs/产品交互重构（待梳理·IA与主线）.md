# 产品交互重构（待梳理 · 信息架构与主线）

> 状态：**方向分析，暂不开发**（用户决策：想清楚先记录）。关联记忆 [[ux-flow-needs-rework]]。
> 触发：后端 V2 内核/记忆已成型，但前端交互沿用 V1 的「层级页面链」，用户**不知如何下手**、业务多为顺序流程、页面下探过深。
> 本篇只定**问题诊断 + 重构方向 + 分期**，不含像素级设计；落地前应配合 [05 产品形态](design/v2/05-产品形态（场景·任务·执行记录·结果·看板）.md) 细化。

## 1. 当前实际交互（grounded，2026-06-27 代码现状）

页面与导航（`frontend/app`）：
```
/(登录) → /dashboard(我的公司：列表 + 新建公司)
        → /orgs/[id](公司主页：花名册展示 + 顶部三按钮[模型配置 / 任务→ / 临时出图→])
            → /orgs/[id]/tasks(任务：新建[名称+目标] → 运行 → 执行记录 → 「查看观测」modal → 展开节点)
            → /orgs/[id]/plans(临时出图：goal → 出图 → 审批 → 运行 → 观测)
            → /orgs/[id]/settings(模型配置)
```
**到「第一个产出」的路径**：登录 → 建/选公司 → 进公司主页 → 点「任务→」→ 新建任务（填名称+目标）→ 运行 → 等 → 点「执行记录」→ 点「查看观测」→ 展开节点。**约 8 步、跨 3~4 个页面层级。**

## 2. 问题诊断

1. **价值不前置 / 入口是「手段」不是「目的」**：登录首屏是「我的公司」列表 + 「新建公司」。但「公司」是手段，用户的**目的是办成一件事**。把「建公司/配角色」摆在主线最前，等于让新用户先理解 Org/Role/Agent 抽象，**还没尝到价值就先扛概念负担**。
2. **顺序流程 = 页面链，缺「工作台」**：每个能力是独立页面（tasks / plans / settings），靠「← 返回」串成线性链。**没有一个「站这儿就能看全局、就能下手」的主操作台**——进了 tasks 看不到 plans，逐层下钻再返回。
3. **下探过深、结果藏太深**：运行后要「执行记录 → 查看观测 → 展开节点」三级点击才看到产出。**结果不是第一公民**（之前修过的观测截断，本质是结构上结果就藏得深的延伸）。
4. **「任务」与「临时出图」并列且语义模糊**：公司页两个入口，用户不知点哪个、区别是什么（一个可复用、一个一次性，UI 没讲清）。
5. **空状态无引导**：没公司/没任务时是空列表，**没有「第一步做什么」+ 示例目标**。用户面对空白输入框，不知道能办什么、该输入什么。
6. **「能办什么」不可见**：公司到底能干哪些事（场景/能力）没有货架式呈现，用户只能凭空想 goal。

> 一句话：**当前是「以公司为中心的层级导航」，用户被迫先理解结构、再逐层下钻；缺一个「以目标为中心、一处下手、结果即见」的入口。**

## 3. 重构方向（核心转变）

**从「以公司为中心的层级导航」→「以目标/任务为中心的工作台」。**

- **A. 首屏 = 目标输入 + 最近/示例**：登录即见「你想办什么？」一个大输入框（起点像对话），下面是最近任务 + 可一键试跑的示例场景。公司退为**背景上下文**（顶部切换器），不是必经第一道门。
- **B. 建公司变成「副产品/一键」**：新用户输入目标 → 「为这个目标开一家公司（按场景自动配角色）」一键完成；**把「建公司+配角色」从前置必做，变成首个任务的自动产物**。专家想手动配再进花名册。
- **C. 统一工作台（一处下手）**：一个 org 工作台页，导航（任务 / 场景库 / 花名册 / 看板 / 设置）在侧/顶，主区默认「任务 + 新建」。**面板切换替代逐页下钻+返回**。
- **D. 「临时出图」收编进任务**：统一为「新建/运行任务」，勾选「保存为可复用任务」与否；消灭并列的第二入口与语义歧义。
- **E. 运行即展开过程 + 结果前置**：运行后在当前位置直接展开「实时过程（节点流）+ 产出」，**结果是第一公民**，不再四级点击。
- **F. 场景库 = 「能办什么」货架**：场景模板按 大类>小类 树浏览（doc 05 §归类），用户可从货架挑、也可自由输入——直接解「不知能干嘛/如何下手」。
- **G. 概念分层呈现**：Org/Role/Agent/Skill/Capability 对新手收进「花名册/高级」；主线只暴露 **目标 → 任务 → 结果**；治理细节（审批/出处/eval/预算）专家再下钻。

## 4. 建议分期（落地时）

1. **一期（最高性价比，解「不知如何下手」）**：① 首屏目标输入框 + 示例/最近；② 运行结果前置（少一两级点击）；③ 空状态引导文案 + 示例目标。**不动数据模型，纯前端 IA 调整。**
2. **二期**：统一工作台（面板化）+ 「临时出图」收编进任务 + 「一键为目标开公司」。
3. **三期**：场景库货架（依赖 doc 05 的 `domain/subcategory` 分类字段）+ 看板（P4）。

## 5. 注意 / 约束
- 这是**产品级重构**，不是小修；动前端前先与本篇 + doc 05 对齐，避免又堆出一层页面链。
- 后端能力已基本就绪（任务/运行/观测/记忆/预算 API 都有），**瓶颈在 IA 与呈现**，不在能力。
- 重构期间遵守 CLAUDE.md：前端改动同样要 DoR/DoD；**勿在没有确认 IA 前擅自大改前端**（见记忆 [[ux-flow-needs-rework]]）。
