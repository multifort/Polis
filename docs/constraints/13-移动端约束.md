# 13 移动端约束（Expo / React Native）

> 覆盖 D1–D9。**适用范围说明**：Polis V1 为 Web 优先；本约束在 Polis 增设移动端时生效，
> 并复用既有 hermes-test 移动端的成熟实践。与 Web 共享设计令牌(10)与类型(B8)。
> 每项：**规则 / 检查 / 能否自动门禁**。

---

## D1 平台与框架基线
- 规则：**Expo SDK(固定大版本) + React Native + TypeScript**；包管理 pnpm/npm 锁定；
  路由 **expo-router**；禁裸 RN 与 Expo 混用导致原生不一致。
- 规则：升级 SDK 走统一流程(锁版本、`expo-doctor` 校验、回归)。
- 检查：`expo-doctor` + lock 校验。**门禁：可**。

## D2 目录结构与导航
- 规则：`app/` 走 expo-router 文件路由；业务按 feature 内聚(`src/features/<域>`)，公共 `src/shared`；
  鉴权分组路由(`(auth)`/`(app)`)，受保护区统一守卫。
- 检查：评审。**门禁：否**。

## D3 组件与样式（与 Web 共享令牌）★
- 规则：**复用 10-UI 的设计令牌**(同一份 tokens 导出 RN theme)；用 `makeStyles(colors)` 或主题 hook，
  禁硬编码颜色/间距；深色模式走令牌(`useColorScheme`/主题 store)，`app.json` 的 `userInterfaceStyle: automatic`。
- 检查：lint(硬编码值) + 评审。**门禁：部分可**。

## D4 状态/数据层
- 规则：与 Web 一致——**服务端态 React Query**，UI 态 Zustand；API client 复用同一套类型(B8，从 OpenAPI 生成)；
  SSE/流式按既有 `expo/fetch` 解析约定(注意 event 行解析、取消异常识别)。
- 检查：评审 + 类型一致。**门禁：部分可**。

## D5 平台适配
- 规则：iOS/Android 双端验证；用 `SafeAreaView`/safe-area；平台差异用 `Platform.select`；
  字体/触达区(hitSlop)/键盘避让统一封装。
- 检查：双端冒烟 + 评审。**门禁：否**。

## D6 性能
- 规则：长列表 `FlatList` 虚拟化 + `keyExtractor`；图片用缓存组件；避免大对象进 state 频繁重渲染；
  动画用原生驱动(`useNativeDriver`)。
- 检查：评审 + 必要时 profiler。**门禁：否**。

## D7 离线 / 网络
- 规则：弱网重试 + 超时；请求失败有可重试错误态(对接 10-A10)；输入草稿不丢失；
  识别取消(Abort/`FetchRequestCanceledException`)，不误报失败。
- 检查：评审 + 弱网测试。**门禁：否**。

## D8 打包 / 发布
- 规则：**EAS Build**；`buildNumber` 每次递增(用工具改并校验，禁手改静默失败)；
  internal(ad-hoc)→ TestFlight 流程固定；OTA(EAS Update) 用于非原生改动；项目路径**纯英文**(避免 CocoaPods 失败)。
- 规则：iOS26+ 用对应 Xcode 镜像；加密合规声明就位。
- 检查：发布 checklist + `eas build:list` 核对。**门禁：否(流程 checklist)**。

## D9 设备能力与权限
- 规则：相机/通知/存储等权限**按需申请 + 文案说明用途**；权限被拒有降级路径；
  敏感能力最小化；遵守平台审核规范。
- 检查：评审 + 权限清单。**门禁：否**。
