# 12 后端风格约束（Python / FastAPI）

> 覆盖 C1–C17。每项：**规则 / 示例(✅正 ❌反) / 检查工具 / 能否自动门禁**。
> 适用：Polis 后端（独立系统）。门禁初期以本地预提交执行（无 CI，见清单注）。

---

## C1 技术栈与版本基线 ★
- 规则：Python **3.12**；FastAPI；SQLAlchemy **2.0**(async)；Pydantic **v2**；Alembic；
  Temporal Python SDK；依赖用 **uv** 或 Poetry 管理，**锁定 lockfile**；运行 ASGI 用 uvicorn。
- 规则：禁止引入与既定栈重复的框架（如再引 Flask/Django、另一套 ORM）。
- 检查：`pyproject.toml` 单一来源；CI/预提交校验 lock 一致。**门禁：可**（lock 校验）。

## C2 项目结构与分层 ★
- 规则：分层 **api → service → repository → domain**，依赖**单向向内**，禁跨层反向依赖。
```text
backend/app/
 ├─ api/        路由(FastAPI router) + 请求/响应 schema(Pydantic)
 ├─ service/    业务用例(编配/规划/路由/执行编排…)
 ├─ repository/ 数据访问(SQLAlchemy)，只此层碰 DB
 ├─ domain/     领域模型/纯逻辑(无框架依赖)
 ├─ integrations/ litellm/temporal/mcp/langfuse/guardrails 适配
 └─ core/       config/logging/security/errors
```
- ❌ 在 `api/` 直接写 SQL；✅ `api`→`service`→`repository`。
- 检查：import-linter 配置层级规则。**门禁：可**（import-linter）。

## C3 命名与编码规范 ★
- 规则：PEP8；模块/函数/变量 `snake_case`，类 `PascalCase`，常量 `UPPER_SNAKE`；
  私有前缀 `_`；包名小写无下划线。文件名与主类语义一致。
- 规则：格式化用 **ruff format**(或 black)，行宽 100；import 排序用 ruff(isort 规则)。
- 检查：`ruff check` + `ruff format --check`。**门禁：可**。

## C4 API 设计规范 ★
- 规则：RESTful；资源名复数(`/agents`,`/orgs`)；动词用 HTTP 方法；嵌套不超 2 层。
- 规则：**统一响应/错误 schema**：
```json
// 错误统一: {"error": {"code":"VALIDATION_ERROR","message":"...","details":[...]}}
```
- 规则：状态码语义正确(200/201/204/400/401/403/404/409/422/500)；分页用 `?limit&cursor`；
  幂等写操作支持 `Idempotency-Key`（创建任务等）。
- 规则：所有路由**声明 response_model**；公开接口走 `/api/v1` 版本前缀。
- 检查：OpenAPI 自动生成 + schemathesis 契约测试。**门禁：可**（契约测试）。

## C5 校验 / 序列化 ★
- 规则：入参/出参**一律 Pydantic v2 模型**，禁手解析 dict；用 `Field` 约束(min/max/regex)。
- 规则：领域配置(Agent/Plan/Preset)用 Pydantic schema 校验后才入库；非法配置返回字段级错误。
- ❌ `body: dict`; ✅ `body: CreateAgentRequest`。
- 检查：mypy + 运行期 Pydantic。**门禁：可**。

## C6 数据库 / ORM 规范 ★
- 规则：只在 `repository/` 用 SQLAlchemy；**禁裸 SQL**，唯一例外是 pgvector 向量检索(集中封装)。
- 规则：表/列 `snake_case`；主键 UUID；所有业务表带 `org_id`(多租户)；时间 `TIMESTAMPTZ`；
  外键显式；**高频查询建索引**(列出 EXPLAIN 依据)；JSONB 字段注明结构。
- 规则：迁移**只走 Alembic**，禁手改库；迁移**向后兼容**、可回滚(见 E9)。
- ❌ `session.execute(text("SELECT ..."))` 散落各处；✅ 集中在 repo + 命名查询。
- 检查：alembic check + repo 层单元测(testcontainers PG)。**门禁：可**。

## C7 异步规范 ★
- 规则：路由/service/repo **全 async**；**禁在 async 中做阻塞 I/O**(同步 requests、time.sleep、阻塞 DB 驱动)；
  阻塞库必须 `run_in_executor` 或换 async 版(httpx/asyncpg)。
- 规则：外部调用(LiteLLM/MCP/HTTP)设 timeout + 重试(指数退避)。
- 检查：ruff 规则(ASYNC*) + 代码评审。**门禁：可**（ruff ASYNC 规则）。

## C8 错误处理与异常 ★
- 规则：定义异常层级 `AppError → (ValidationError|NotFound|Forbidden|ConflictError|UpstreamError…)`；
  FastAPI 全局 exception handler 统一转 C4 错误 schema。
- 规则：**禁裸 `except:`**；禁吞异常(`except: pass`)；捕获要么处理要么带上下文重抛。
- ❌ `try: ... except: pass`；✅ `except UpstreamError as e: raise ... from e`。
- 检查：ruff(BLE/E722) + 评审。**门禁：可**。

## C9 日志与可观测 ★
- 规则：**结构化日志**(structlog/JSON)；每请求注入 `correlation_id`(贯穿 service→repo→外部调用)；
  关键路径(规划/路由/执行/模型调用)上报 **Langfuse trace**；禁 `print`。
- 规则：日志**严禁出现密钥/凭证/PII**(见 C10/C16)。
- 检查：ruff(禁 print) + 评审 + 日志脱敏单测。**门禁：部分可**。

## C10 配置与密钥 ★
- 规则：配置用 **pydantic-settings** 从环境读，集中 `core/config.py`，禁散落 `os.getenv`；
  **密钥永不入代码/仓库/日志/上下文/记忆**；用户 BYO Key 走 Credential Broker(信封加密、短时注入)。
- 规则：`.env.example` 提供模板，`.env` 进 .gitignore；secret 扫描(见 E14)。
- 检查：gitleaks/secret 扫描 + 评审。**门禁：可**（secret 扫描）。

## C11 鉴权与多租户 ★
- 规则：所有受保护路由经 auth 依赖注入(JWT)；**每个查询强制 `org_id` 过滤**，
  service 层统一注入租户上下文，禁手写时遗漏；敏感操作走 RBAC(owner/approver/member)。
- 规则：跨租户访问**默认拒绝**；提供"租户隔离回归测试"(见 E17)。
- ❌ `repo.list_agents()` 无 org 过滤；✅ `repo.list_agents(org_id=ctx.org_id)`。
- 检查：隔离回归测试 + 评审。**门禁：可**（隔离测试）。

## C12 Temporal 规范 ★
- 规则：Workflow 内**必须确定性**——禁直接 I/O、随机、`datetime.now()`、读环境；
  这些只能在 **Activity** 里做。Workflow 只编排。
- 规则：Activity **幂等**(配 Idempotency-Key/去重)；设 `start_to_close_timeout` + RetryPolicy；
  人审用 signal/`wait_condition`；长流程注意 history 大小(必要时 continue-as-new)。
- ❌ workflow 内 `await db.query()`；✅ `await workflow.execute_activity(run_node, ...)`。
- 检查：Template 评审清单 + 重放测试(workflow replay test)。**门禁：可**（replay test）。

## C13 LLM / Agent 调用规范 ★
- 规则：**所有模型调用只经 ModelGateway(LiteLLM)**，禁业务代码直连 OpenAI/Claude SDK；
  统一带 token 预算、超时、重试、Langfuse metadata；embedding/rerank 同样经网关。
- 规则：工具调用只经 MCP 运行时；外部内容回流先过 Guardrails(见 C16/04)。
- 检查：禁止 import 厂商 SDK 的 ruff/自定义规则 + 评审。**门禁：可**（import 禁用规则）。

## C14 测试规范 ★
- 规则：**pytest + pytest-asyncio**；分层——domain/service 单元测、repo 用 **testcontainers PG** 集成、
  api 用 httpx AsyncClient、Temporal 用 test env。关键路径必须有测试。
- 规则：用 fixture/factory 造数据；禁测试间共享可变状态；外部(LiteLLM/MCP)用录制或 mock。
- 规则：覆盖率门槛见 E10（核心模块 ≥ 指定阈值）。
- 检查：`pytest` + `coverage`。**门禁：可**。

## C15 类型注解
- 规则：公共函数**全类型注解**；`mypy` 严格(`disallow_untyped_defs`)；禁 `Any` 滥用(必要处加注释说明)。
- 检查：`mypy`。**门禁：可**。

## C16 后端安全 ★
- 规则：所有外部输入校验(Pydantic)；**注入纵深防御**(Guardrails 过滤进 Agent 的外部内容，见 04)；
  危险动作(发布/外发/支付/删除)强制人审 gate；依赖 CVE 扫描(见 E14)；SQL 仅 ORM 参数化。
- 规则：审计日志记录敏感操作(谁/何时/对象)。
- 检查：bandit(SAST) + pip-audit + 评审。**门禁：可**。

## C17 性能与资源
- 规则：DB 连接池配置上限；**禁 N+1**(用 join/selectinload)；热点查询缓存(谨慎，先量再缓)；
  每 Agent/任务设 `max_tokens`/`max_tool_calls`/并发上限(对应 Agent policy)。
- 检查：慢查询日志 + 评审 + 必要时压测(见 E15)。**门禁：部分(慢查询告警)**。

## 门禁汇总（初期本地预提交，CI 引入后自动化）
```text
pre-commit: ruff check + ruff format --check + mypy + bandit + gitleaks
test 门槛:  pytest + coverage(E10 阈值) + 租户隔离测试 + temporal replay test
评审清单:   分层依赖 / 无裸except / 只经网关调模型 / org_id 过滤 / 迁移向后兼容
```
