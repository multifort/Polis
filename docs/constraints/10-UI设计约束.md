# 10 UI / 设计约束（跨端设计系统）

> 覆盖 A1–A14。每项：**规则 / 示例 / 检查方式 / 能否自动门禁**。
> 这套设计令牌是前端(11)/移动端(13)样式的**唯一来源**，先定这里再落代码。

---

## A1 设计令牌(Design Tokens) ★
- 规则：所有视觉值**只来自令牌**，禁散落魔法值。令牌分三层：
  **基础令牌**(原始值) → **语义令牌**(role: bg/surface/text/primary/danger…) → **组件令牌**。
- 规则：令牌用平台无关格式(JSON/Style Dictionary)，导出到 Tailwind theme(web) 与 RN theme(移动)。
```jsonc
{ "color": { "primary":{"value":"#2563eb"}, "danger":{"value":"#dc2626"} },
  "space": { "1":"4px","2":"8px","3":"12px","4":"16px" },
  "radius":{ "sm":"6px","md":"10px" } }
```
- 检查：组件中禁硬编码颜色/间距(lint 规则) + 设计评审。**门禁：部分可**。

## A2 主题与深色模式 ★
- 规则：light/dark **两套语义令牌映射**，组件只引语义令牌(`bg`,`text`,`border`)，切换主题不改组件；
  支持"跟随系统/浅/深"三档。深色不是简单反色，单独调色板。
- 检查：双主题快照测试 + 评审。**门禁：部分可(快照)**。

## A3 排版与字体 ★
- 规则：定义**字号阶梯**(如 12/13/14/16/20/24/28)、行高、字重；中文优先 PingFang/思源黑体、等宽 Menlo/monospace；
  正文 14、说明 12、标题层级固定。禁随意字号。
- 检查：令牌约束 + 评审。**门禁：否**。

## A4 配色与语义色 ★
- 规则：品牌主色 + **语义色**(success/warning/danger/info)固定；状态(成功/失败/进行中)用语义色，禁自定义；
  保证对比度(见 A11)。
- **★ 具体色值规范见 [10a 色彩规范](./10a-色彩规范.md)**：它是颜色的**唯一来源**(基础/语义令牌、对比度实测、淡底派生规则、深色映射)；
  本项只定原则，所有十六进制与令牌以 10a 为准，组件禁硬编码。
- 检查：评审 + 对比度检查。**门禁：部分可**。

## A5 间距与栅格
- 规则：间距用 4px 基数阶梯(4/8/12/16/24/32…)；布局 12 栅格 + 断点；卡片/列表统一内距。
- 检查：lint(间距魔法值) + 评审。**门禁：部分可**。

## A6 组件库规范 ★
- 规则：建**统一组件库**(shared/ui)，每个组件定义：变体(variant)、尺寸(size)、**全状态**(default/hover/active/disabled/loading/error)；
  业务页面**只能用组件库 + 令牌**拼装，禁页面内重复造基础组件。组件文档化(Storybook 可选)。
- 规则：组件清单基线：Button/Input/Select/Textarea/Checkbox/Radio/Switch/Modal/Drawer/Toast/Tooltip/Tabs/Table/Card/Badge/Tag/Avatar/Skeleton/EmptyState/Pagination/Breadcrumb/Form。
- 检查：评审 + (可选)Storybook 视觉测试。**门禁：否**。

## A7 图标与插画
- 规则：单一图标库(如 lucide)，统一尺寸/线宽/颜色(令牌)；禁混用多套图标；插画风格统一。
- 检查：评审。**门禁：否**。

## A8 交互与动效
- 规则：过渡时长令牌化(fast 120ms/base 200ms/slow 320ms)、统一缓动；加载用 **skeleton/spinner** 规范；
  禁过度动画影响性能/可达性(尊重 reduce-motion)。
- 检查：评审。**门禁：否**。

## A9 表单与校验展示 ★
- 规则：统一**内联校验**：失焦/提交触发，错误显示在字段下方(语义 danger 色 + 文案)；必填标记统一；
  提交态禁用按钮 + loading；成功/失败统一反馈(见 A10)。
- ❌ 仅 alert 弹"输入有误"；✅ 字段级、具体、内联。
- 检查：评审 + 组件库强制。**门禁：否**。

## A10 反馈与状态 ★
- 规则：四态统一组件——**加载(skeleton/spinner)、空态(EmptyState+引导)、错误态(可重试)、成功(toast)**；
  危险操作二次确认(Modal)；长任务进度可见(对接公司看板 A13)。
- 检查：页面评审清单(四态齐全)。**门禁：否**。

## A11 无障碍(a11y)
- 规则：文本对比度 ≥ WCAG AA(正文 4.5:1)；可键盘操作 + 焦点可见；交互元素有 aria-label；图片 alt。
- 检查：axe/eslint-jsx-a11y + Lighthouse。**门禁：可(a11y lint)**。

## A12 国际化/多语言
- 规则：**文案不硬编码**，走 i18n key(即便初期只中文)；日期/数字/货币用 locale 格式化；预留 RTL 不破坏布局。
- 检查：lint(禁硬编码中文字面量于组件，按需) + 评审。**门禁：部分可**。

## A13 数据可视化 / 看板规范 ★
- 规则：图表库统一(如 Recharts/visx)；图表配色用语义令牌；**公司看板**统一卡片：
  任务状态(进行/等待/失败/完成)、当前角色/节点、已花成本、卡点；表格统一(排序/分页/空态/密度)。
- 规则：状态色与 A4 语义色一致(成功绿/失败红/进行蓝)，全局一致。
- 检查：评审 + 组件库。**门禁：否**。

## A14 文案与术语规范
- 规则：建**统一术语表**(Org=公司/工作区、Agent、角色、计划、技能、审批…)，全端一致；
  语气简洁、动词开头；错误文案给"原因+下一步"。
- 检查：术语表评审 + 文案 review。**门禁：否**。

## 落地与门禁
```text
唯一来源: tokens(JSON/Style Dictionary) → web(Tailwind theme) / 移动(RN theme)
可自动:   禁硬编码颜色/间距 lint · a11y lint · 双主题快照
靠评审:   组件库唯一性 · 四态齐全 · 看板/术语一致
```
