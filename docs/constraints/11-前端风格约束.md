# 11 前端风格约束（React / Next.js + TypeScript）

> 覆盖 B1–B14。每项：**规则 / 示例(✅正 ❌反) / 检查工具 / 能否自动门禁**。
> 适用：Polis Web 前端（新建，独立系统）。

---

## B1 技术栈与版本基线 ★
- 规则：**Next.js(App Router) + React 18+ + TypeScript**；包管理 **pnpm**(锁 lockfile)；Node LTS。
- 规则：禁混入第二套同类方案(如再引 CRA/Vite 另起、两套状态库)。
- 检查：`package.json` 单源 + lock 校验。**门禁：可**。

## B2 目录结构与分层 ★
- 规则：**feature-based**，按业务域聚合，公共下沉 `shared/`。
```text
src/
 ├─ app/                Next 路由(页面/布局)
 ├─ features/<域>/       components/ hooks/ api/ types/  (org, planning, approval, dashboard…)
 ├─ shared/             ui(组件库) lib(工具) api(client) types hooks
 └─ styles/             tokens/全局
```
- ❌ 把所有组件堆 `components/`；✅ 按 feature 内聚。
- 检查：eslint import 边界规则。**门禁：可**。

## B3 命名规范 ★
- 规则：组件文件/组件名 `PascalCase`；hook `useXxx`；普通工具 `camelCase`；
  类型/接口 `PascalCase`(禁 `I` 前缀)；常量 `UPPER_SNAKE`；目录 `kebab-case`。
- 检查：eslint(naming) + 评审。**门禁：部分可**。

## B4 组件规范 ★
- 规则：**函数组件 + hooks**，禁 class 组件；props **显式 TS 类型**(禁 `any`)；
  组件保持单一职责、可组合；展示与逻辑分离(容器/展示或 hooks 抽逻辑)。
- 规则：禁在 render 内创建新函数/对象导致无谓重渲染(必要处 `useCallback`/`useMemo`)。
- ❌ `function C(props: any)`；✅ `function C({ id, title }: CProps)`。
- 检查：eslint(react-hooks) + tsc。**门禁：可**。

## B5 状态管理 ★
- 规则：**服务端状态用 TanStack Query(React Query)**(缓存/重试/失效)，**禁用全局 store 存服务端数据**；
  客户端 UI 状态用 **Zustand**(轻量)或局部 `useState`；禁 prop drilling 超 2 层(用 context/store)。
- ❌ 用 Redux 存接口返回的列表；✅ React Query 管服务端、Zustand 管 UI 态。
- 检查：评审 + 约定。**门禁：否(评审)**。

## B6 路由规范
- 规则：用 Next **App Router**；路由段语义化；服务端组件优先、交互处用 `"use client"`；
  受保护路由统一鉴权(middleware/layout 守卫)。
- 检查：评审。**门禁：否**。

## B7 样式方案 ★
- 规则：**Tailwind CSS + 设计令牌**(令牌来自 10-UI，映射成 Tailwind theme)；禁内联魔法值(颜色/间距用令牌)；
  复杂组件可 CSS Module 辅助；深色模式走令牌(见 10-A2)。
- ❌ `style={{color:'#1e90ff'}}`；✅ `className="text-primary"`(令牌)。
- 检查：eslint(禁内联 style 魔法值规则) + 评审。**门禁：部分可**。

## B8 API / 数据层 ★
- 规则：统一 **api client**(fetch 封装：baseURL、鉴权头、错误归一化)；**类型从后端 OpenAPI 生成**(openapi-typescript)，
  禁手抄类型；所有请求经 React Query hook，集中错误/加载态处理。
- ❌ 组件里直接 `fetch`；✅ `useAgents()`(React Query + 生成类型)。
- 检查：生成类型校验 + 评审。**门禁：可(类型生成校验)**。

## B9 类型安全 ★
- 规则：`tsconfig` **strict: true**；**禁 `any`**(必要处 `unknown` + 收窄并注释)；禁 `@ts-ignore`(用 `@ts-expect-error` 且注明)。
- 检查：`tsc --noEmit` + eslint(no-explicit-any)。**门禁：可**。

## B10 表单方案
- 规则：表单用 **React Hook Form + Zod**(schema 校验，与后端约束对齐)；统一错误展示(见 10-A9)。
- 检查：评审。**门禁：否**。

## B11 性能规范
- 规则：路由级代码分割(动态 import)；列表大数据虚拟化；图片用 `next/image`；
  避免无谓重渲染(memo/选择器)；首屏关键数据 SSR/预取。
- 检查：Lighthouse/bundle 分析(按需) + 评审。**门禁：否**。

## B12 Lint / 格式化 ★
- 规则：**ESLint(typescript-eslint + react + react-hooks + import) + Prettier**；规则集纳入仓库；保存即格式化。
- 检查：`eslint .` + `prettier --check`。**门禁：可**。

## B13 测试规范
- 规则：**Vitest + Testing Library**(组件/hook 单元)；关键流程 **Playwright** e2e(登录/建公司/计划审批)；
  测交互与可达性，禁测实现细节。
- 检查：`vitest` + `playwright`。**门禁：可**。

## B14 前端安全 ★
- 规则：**禁 `dangerouslySetInnerHTML`**(必须时严格清洗 DOMPurify)；token 用 httpOnly cookie 或安全存储，
  **禁存 localStorage 明文长期 token**；env 仅 `NEXT_PUBLIC_*` 暴露给前端；依赖 CVE 扫描(E14)。
- 检查：eslint(react/no-danger) + 评审 + npm audit。**门禁：可**。

```text
test:       vitest(coverage E10) + playwright(关键流)

```
